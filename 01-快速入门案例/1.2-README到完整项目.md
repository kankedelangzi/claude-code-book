# 1.2 从 README 到完整项目

> 真实场景：拿到一个 GitHub 项目，从零跑起来。

## 🎯 场景描述

你刚接手一个新项目，仓库里只有一个 `README.md`，写着：

> 一个基于 FastAPI 的待办事项 API，支持用户认证、任务 CRUD、标签分类。

但代码呢？没有。你需要从零开始搭建。

**传统做法：**
1. 搭项目骨架（30 分钟）
2. 装依赖、配环境（20 分钟）
3. 写数据模型（40 分钟）
4. 写 API 路由（1 小时）
5. 写认证逻辑（40 分钟）
6. 写测试（30 分钟）

**预计耗时：3-4 小时**

## ✨ Claude Code 做法

### 第一步：初始化项目

打开终端，进入项目目录，启动 Claude Code：

```bash
cd todo-api
claude
```

输入你的第一条指令：

```
根据 README 的描述，帮我搭建完整项目：

技术栈：
- Python 3.11 + FastAPI
- SQLite 数据库 + SQLAlchemy ORM
- JWT 认证
- pytest 测试

功能需求：
1. 用户注册/登录（JWT token）
2. 待办事项 CRUD
3. 标签分类
4. 每个待办可以挂多个标签

请按标准项目结构组织代码，包含 requirements.txt。
```

### 第二步：看它怎么工作

Claude Code 会自动创建：

```
todo-api/
├── app/
│   ├── __init__.py
│   ├── main.py          # FastAPI 入口
│   ├── database.py      # 数据库连接
│   ├── models.py         # SQLAlchemy 模型
│   ├── schemas.py        # Pydantic 模式
│   ├── auth/
│   │   ├── __init__.py
│   │   ├── router.py     # 登录/注册接口
│   │   └── utils.py      # JWT 工具
│   └── todos/
│       ├── __init__.py
│       └── router.py     # 待办事项接口
├── tests/
│   ├── test_auth.py
│   └── test_todos.py
├── requirements.txt
└── README.md
```

### 第三步：运行测试

```
帮我运行测试，看看有没有问题。
```

Claude Code 会执行 `pytest` 并修复发现的问题。

### 第四步：补充细节

```
给 API 加上 Swagger 文档描述，每个接口加上中文说明。
再加一个分页查询接口 GET /todos?page=1&size=20。
```

## 💡 进阶技巧

### 分步推进，不要一步到位

与其一口气丢一个大需求，不如分步：

```
# 先搭骨架
帮我初始化 FastAPI 项目结构，只用 SQLite，先不加认证。

# 再加功能
现在加上 JWT 认证，用 python-jose。

# 最后补测试
给所有接口写 pytest 测试。
```

**好处**：每一步都可以检查、调整，避免后面推翻重来。

### 善用"改一下"指令

```
把 Todo 模型加一个 priority 字段，1-3 优先级，默认 2。
同时更新对应的 schema 和 API。
```

Claude Code 会自动找到所有相关文件并修改。

## ⚠️ 注意事项

1. **检查生成的依赖版本**——有时候会指定过旧的版本
2. **认证逻辑要自己审计**——安全相关的东西不要盲目信任
3. **项目结构可以调整**——如果不喜欢它的组织方式，直接说"按 xxx 结构重新组织"
4. **SQL 文件要自己检查**——ORM 生成的迁移可能不完美

## 📝 小结

从 README 到完整项目，本质上是把**结构化的需求**变成**可运行的代码**。Claude Code 最擅长这种事——你提供想法，它负责搬砖。关键是**分步推进**，每步检查，别贪多。
