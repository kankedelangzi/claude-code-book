# 1.1 五分钟体验 Claude Code

> 本章带你快速感受 Claude Code 的威力，用一个真实案例让你立刻上手。

## 🎯 场景描述

假设你收到一个任务：**给公司内部系统写一个用户管理 API**。

传统做法：
1. 设计数据结构
2. 写 SQL 建表
3. 写 API 接口
4. 写测试用例
5. 写文档

**预计耗时：2-3 小时**

## ✨ Claude Code 做法

你只需要在项目目录打开 Claude Code，输入：

```
帮我创建一个用户管理模块：
- 用户表包含 id, name, email, created_at
- 提供 CRUD API（增删改查）
- 用 Node.js + Express
- 带基础测试用例
```

**Claude Code 会自动：**
1. 创建 `models/User.js`
2. 创建 `routes/users.js`
3. 创建 `tests/user.test.js`
4. 写好完整代码

**预计耗时：5-10 分钟**

## 💡 关键技巧

### 技巧 1：需求要具体

❌ 模糊的需求：
```
帮我写个用户管理
```

✅ 清晰的需求：
```
帮我创建一个用户管理模块：
- 用户表包含 id, name, email, created_at
- 提供 CRUD API（增删改查）
- 用 Node.js + Express
- 带基础测试用例
```

**原因**：Claude Code 需要明确的上下文才能生成你想要的代码。

### 技巧 2：一步一步来

不要一次性丢一个巨大的需求，分步推进：

```
# 第一步：搭骨架
帮我初始化 Express 项目，只要基础结构

# 第二步：加功能
现在添加用户管理 API，包含 CRUD

# 第三步：补测试
给用户 API 写测试用例

# 第四步：加验证
给 API 加参数校验，用 express-validator
```

**好处**：
- 每一步都能检查结果
- 发现问题可以及时调整
- 避免一次性生成大量错误代码

### 技巧 3：让它解释代码

```
解释一下这个 User 模型的代码逻辑
```

Claude Code 会详细解释，帮你理解生成的代码。

### 技巧 4：让它改，不要自己改

❌ 自己打开文件改：
```
（你打开 models/User.js，手动加了一个字段）
```

✅ 让 Claude Code 改：
```
给 User 模型加一个 role 字段，可选值是 'admin' | 'user'，默认 'user'
```

**原因**：
- Claude Code 会同步更新所有相关文件（模型、路由、测试）
- 你手动改容易漏掉关联文件

## ⚠️ 注意事项

### 1. 生成的代码要 review

Claude Code 生成的代码**不是完美的**，你需要检查：

- 数据库配置是否正确
- 安全验证是否完善
- 错误处理是否合理

### 2. 安全相关要自己审计

认证、授权、加密等安全相关的代码，**务必自己审计**：

```
检查一下这个 JWT 认证实现有没有安全问题
```

### 3. 依赖版本要检查

Claude Code 可能会指定过旧的依赖版本：

```json
{
  "dependencies": {
    "express": "^4.17.1"  // 可能不是最新版
  }
}
```

检查并更新到合适的版本。

### 4. 项目结构可以调整

如果不喜欢 Claude Code 生成的项目结构，直接说：

```
我不喜欢这个结构，按 MVC 模式重新组织
```

## 🎓 进阶：让它帮你学习

### 问它为什么

```
为什么用 bcrypt 而不是直接存密码？
```

Claude Code 会解释，你顺便学到了安全知识。

### 问它替代方案

```
除了 JWT，还有什么认证方式？各有什么优缺点？
```

### 问它最佳实践

```
这个用户管理 API 有什么可以优化的地方？
```

## 📝 小结

这就是 Claude Code 的核心价值：

> **把繁琐的重复劳动交给 AI，你专注在业务逻辑和创意设计。**

记住三个要点：
1. **需求要具体**——给它清晰的上下文
2. **分步推进**——不要一次性丢大需求
3. **代码要 review**——它生成的不是完美的
