# 1.3 让 Claude Code 帮你写第一行代码

> 不会编程？没关系。让 Claude Code 做你的编程老师。

## 🎯 场景描述

假设你：

- 知道要做什么，但不会写代码
- 或者会一点，但不确定怎么开始
- 或者很久没写代码，手生了

你的需求：**写一个 Python 脚本，批量重命名文件**。比如把 `IMG_20240101_001.jpg` 改成 `2024-01-01-001.jpg`。

## ✨ Claude Code 做法

### 不需要知道怎么写，只需要说清楚要做什么

```bash
claude
```

```
帮我写一个 Python 脚本：

功能：批量重命名当前目录下的 JPG 图片
规则：
- 文件名格式为 IMG_YYYYMMDD_编号.jpg
- 改成 YYYY-MM-DD-编号.jpg
- 比如 IMG_20240101_001.jpg → 2024-01-01-001.jpg
- 只处理 .jpg 文件，不递归子目录
- 先预览要改什么，确认后再执行
```

Claude Code 会生成类似这样的脚本：

```python
#!/usr/bin/env python3
"""批量重命名 JPG 图片文件名。"""

import os
import re
from datetime import datetime

def parse_filename(filename):
    """解析旧文件名，返回新文件名。"""
    match = re.match(r'IMG_(\d{4})(\d{2})(\d{2})_(\d+)\.jpg', filename, re.IGNORECASE)
    if not match:
        return None
    year, month, day, num = match.groups()
    return f"{year}-{month}-{day}-{num}.jpg"

def preview_renames(directory):
    """预览重命名结果。"""
    renames = []
    for f in sorted(os.listdir(directory)):
        new_name = parse_filename(f)
        if new_name and new_name != f:
            renames.append((f, new_name))
    return renames

def main():
    directory = os.getcwd()
    renames = preview_renames(directory)

    if not renames:
        print("没有需要重命名的文件。")
        return

    print(f"找到 {len(renames)} 个文件待重命名：\n")
    for old, new in renames:
        print(f"  {old}  →  {new}")

    confirm = input(f"\n确认执行？(y/N): ").strip().lower()
    if confirm != 'y':
        print("已取消。")
        return

    for old, new in renames:
        old_path = os.path.join(directory, old)
        new_path = os.path.join(directory, new)
        os.rename(old_path, new_path)
        print(f"  ✅ {old} → {new}")

    print(f"\n完成！共重命名 {len(renames)} 个文件。")

if __name__ == '__main__':
    main()
```

### 你能学到什么

这段代码教会你：

| 知识点 | 说明 |
|--------|------|
| `os.listdir()` | 列出目录内容 |
| `re.match()` | 正则表达式匹配 |
| `os.rename()` | 重命名文件 |
| `input()` | 等待用户确认 |
| `if __name__ == '__main__'` | Python 入口写法 |

**这比看教程快多了**——你拿到的是能用的代码，而且是你自己描述的需求。

## 🔧 进阶：让代码更健壮

拿到初版后，你可以继续让 Claude Code 改进：

```
给脚本加上这些改进：
1. 支持命令行参数指定目录，默认当前目录
2. 支持 --dry-run 参数只预览不执行
3. 文件名冲突时自动加后缀
4. 加上 undo 功能，记录操作日志，支持撤销
5. 支持 .jpeg 后缀
```

Claude Code 会逐步完善代码，你只需要提需求。

## 💡 不会编程者的使用建议

### 1. 用自然语言描述需求

你不需要知道函数叫什么、语法怎么写。只需要说清楚：

```
我想做什么 → 什么格式 → 什么规则 → 什么限制
```

### 2. 先跑通，再理解

```
运行成功后，帮我给每行代码加上中文注释。
```

### 3. 出错了直接贴

```
运行报错了：
Traceback (most recent call last):
  File "rename.py", line 15, in <module>
    ...
帮我修复。
```

### 4. 不会问，就问 Claude Code

```
这个正则表达式 r'IMG_(\d{4})...' 是什么意思？
逐个给我解释。
```

## ⚠️ 注意事项

1. **别把 AI 当黑盒**——至少看看生成的代码在干什么
2. **先在测试目录跑**——批量操作文件前，先复制几份测试
3. **脚本要自己保存**——Claude Code 的会话不会永久保存你的文件
4. **逐步加功能**——别一次要太多，容易出错

## 📝 小结

"写第一行代码"不是比喻——你真的不需要会写代码。Claude Code 让你用**自然语言**编程，用**对话**迭代。你的编程能力 = **描述问题的能力**。

不会写代码不是障碍。不知道要做什么才是。
