## 🔧 常见报错速查

| 报错类型 | 最常见原因 | 快速修复 |
|---------|-----------|---------|
| `undefined is not a function` | 函数未定义或拼写错误 | 检查 import 是否正确 |
| `Cannot read property 'X' of null` | 对象是 null | `obj?.X` 或默认值 |
| `Maximum update depth exceeded` | useEffect 依赖数组有问题 | 添加空数组 `[]` 或正确依赖 |
| `CORS error` | 跨域请求未配置 | 后端加 CORS 头或代理 |
| `404 on localhost` | 路径拼写或端口错误 | 检查服务是否启动 |

## 🚀 进阶案例：复杂报错链路追踪

### 案例：React + TypeScript 项目报错

**报错信息：**

```
TypeError: Cannot read properties of undefined (reading 'filter')
    at UserList (src/components/UserList.tsx:42:18)
    at renderWithHooks (react-dom.development.js:14985:18)
    at mountIndeterminateComponent (react-dom.development.js:17802:12)
```

**给 Claude Code 的完整信息：**

```
报错了，帮我分析：

报错信息：
```
TypeError: Cannot read properties of undefined (reading 'filter')
    at UserList (src/components/UserList.tsx:42:18)
```

技术栈：React 18 + TypeScript + Redux Toolkit

组件代码（第40-50行）：
```tsx
const UserList = () => {
  const users = useAppSelector(state => state.users.list)
  const filtered = users.filter(u => u.active) // 第42行
  return <ul>{filtered.map(u => <li key={u.id}>{u.name}</li>)}</ul>
}
```

Redux slice：
```tsx
const usersSlice = createSlice({
  name: 'users',
  initialState: { list: [], loading: false },
  reducers: { ... }
})
```

问题：页面刷新后偶现这个报错，刷新几次又好了。
```

**Claude Code 分析结果：**

```
问题定位：
1. users 在某时刻是 undefined，不是空数组 []
2. 偶现说明和异步数据加载有关

可能原因：
1. Redux initialState 设置了 list: []，但某处代码把它设成了 undefined
2. 组件渲染时，Redux 状态还没初始化完成
3. 某个 reducer 错误地返回了 undefined

建议检查：
1. 搜索所有对 state.users.list 的赋值
2. 检查 RTK Query 或异步 action 的处理
3. 加防御代码：const users = useAppSelector(state => state.users.list) ?? []
```

### 案例：Node.js 后端报错

**报错信息：**

```
Error: Cannot set headers after they are sent to the client
    at ServerResponse.setHeader (_http_outgoing.js:485:11)
    at res.header (express/lib/response.js:769:10)
```

**给 Claude Code 的分析请求：**

```
这个报错是什么意思？代码如下：

```javascript
router.post('/order', async (req, res) => {
  const order = await createOrder(req.body)
  res.json({ success: true, order })
  
  // 发送通知（不阻塞响应）
  sendNotification(order.id).catch(console.error)
  
  // 记录日志
  logOrder(order.id)
  res.json({ logged: true }) // 这里报错
})
```
```

**Claude Code 分析：**

```
问题：res.json() 被调用了两次

分析：
1. 第一个 res.json() 已经发送响应并结束
2. 第二个 res.json() 尝试再次发送，报错

修复方案：
```javascript
router.post('/order', async (req, res) => {
  const order = await createOrder(req.body)
  
  // 异步操作放后面，不要再调 res.json
  sendNotification(order.id).catch(console.error)
  logOrder(order.id)
  
  // 只发送一次响应
  res.json({ success: true, order })
})
```
```

## 🔍 报错信息解读速查表

### JavaScript/TypeScript 常见报错

| 报错信息 | 原因分析 | 快速修复 |
|---------|---------|---------|
| `undefined is not a function` | 变量是 undefined 但当函数调用 | 检查变量定义和 import |
| `Cannot read property 'X' of undefined` | 对象为 undefined | `obj?.X` 或 `obj \|\| {}` |
| `Cannot read property 'X' of null` | 对象为 null | 加 null 检查 |
| `X is not defined` | 变量未声明 | 检查作用域和拼写 |
| `Maximum call stack size exceeded` | 无限递归 | 检查递归终止条件 |
| `Unexpected token` | 语法错误 | 检查括号、引号匹配 |

### React 常见报错

| 报错信息 | 原因分析 | 快速修复 |
|---------|---------|---------|
| `Maximum update depth exceeded` | 无限重渲染 | 检查 useEffect 依赖数组 |
| `Objects are not valid as a React child` | 直接渲染对象 | 渲染对象的具体属性 |
| `Cannot update a component while rendering` | 渲染时调用 setState | 把 setState 移到 useEffect |
| `Each child in a list should have a unique "key"` | 列表缺少 key | 加 `key={item.id}` |

### Node.js 常见报错

| 报错信息 | 原因分析 | 快速修复 |
|---------|---------|---------|
| `ECONNREFUSED` | 连接被拒绝 | 检查服务是否启动 |
| `ETIMEDOUT` | 连接超时 | 检查网络或增加超时时间 |
| `Cannot set headers after they are sent` | 重复发送响应 | 一个请求只发一次响应 |
| `Port 3000 is already in use` | 端口被占用 | 换端口或杀掉占用进程 |
