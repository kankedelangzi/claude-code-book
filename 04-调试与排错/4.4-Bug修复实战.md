# 4.4 Bug 修复实战

> 让 Claude Code 帮你一步步定位和修复 Bug。

## 🎯 场景描述

用户报告：订单页面点击"确认下单"后没有反应。

你打开代码，看了几分钟，发现逻辑有点复杂：

```javascript
async function handleSubmit() {
  // 1. 校验表单
  if (!validateForm()) {
    showError('请填写完整')
    return
  }

  // 2. 提交订单
  const result = await submitOrder(formData)
  
  // 3. 更新库存
  await updateInventory(orderId)
  
  // 4. 发送通知
  sendNotification(userId, orderId)
  
  // 5. 跳转页面
  navigateTo('/success')
}
```

不知道是哪一步出了问题。

## ✨ Claude Code 做法

**第一步：缩小范围**

```
我有个 Bug：点击确认下单后没反应。请帮我分析最可能的原因：

代码逻辑是：
[粘贴 handleSubmit 函数]

可能的问题：
- 校验失败但没提示？
- API 调用失败？
- 页面跳转逻辑有问题？
```

**第二步：加调试代码**

```
在 handleSubmit 里加 console.log，帮我定位到哪一步卡住了
```

**第三步：修复问题**

```
问题定位到了，是因为 updateInventory 返回错误但没 catch。
帮我修复这个问题，添加错误处理。
```

**第四步：验证修复**

```
帮我写一个测试用例，模拟 updateInventory 失败的情况，
确保错误被正确处理而不是静默失败。
```

## 📂 完整对话流程

```
你：Bug 反馈：点击下单没反应，帮我看看
Claude：分析代码后，列出几个可能原因，按概率排序：
       1. validateForm() 某字段校验失败
       2. updateInventory() 抛异常
       3. 表单数据格式不对

你：可能是 updateInventory 的问题
Claude：好的，我来分析这个函数的实现...
       发现问题了：updateInventory 在网络错误时没有正确处理。
       建议加 try-catch。

你：帮我加上，并确保错误时用户能看到提示
Claude：[生成修复代码]

你：完美，帮我加个测试用例验证
Claude：[生成测试]
```

## 💡 Bug 修复技巧

**高效报告 Bug 给 Claude Code：**

```
Bug：订单提交后金额显示错误
期望：显示 99.99
实际：显示 99.99000000000001
技术栈：React + TypeScript
复现步骤：1. 选择商品 2. 输入价格 3. 提交
相关代码：[粘贴]
```

**分步修复比一步到位更靠谱：**

1. 先问"可能原因是什么"
2. 确认方向后问"怎么定位"
3. 定位后问"怎么修复"
4. 最后问"怎么测试"

**偶现 Bug 的处理：**

```
这个 Bug 不是每次都出现，可能和并发/异步有关。
帮我检查所有异步操作的顺序和错误处理。
```
