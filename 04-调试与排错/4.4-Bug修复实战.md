## 🚀 进阶案例：复杂 Bug 修复

### 案例 1：状态不同步问题

**问题描述：**

```
Bug：购物车数量和实际不一致
复现：
1. 添加商品 A 到购物车，显示数量 1
2. 添加商品 A 到购物车，显示数量 1（应该是 2）
3. 刷新页面，显示数量 2

技术栈：React + Redux Toolkit
```

**让 Claude Code 分析：**

```
购物车数量不同步，添加两次但 UI 只显示 1，刷新后才正确。

Redux slice：
```javascript
const cartSlice = createSlice({
  name: 'cart',
  initialState: { items: [] },
  reducers: {
    addItem: (state, action) => {
      const existing = state.items.find(i => i.id === action.payload.id)
      if (existing) {
        existing.quantity += 1
      } else {
        state.items.push({ ...action.payload, quantity: 1 })
      }
    }
  }
})
```

组件：
```javascript
const Cart = () => {
  const items = useAppSelector(state => state.cart.items)
  const total = items.reduce((sum, i) => sum + i.quantity, 0)
  return <span>{total}</span>
}
```

请分析可能原因。
```

**Claude Code 分析：**

```
问题定位：
Redux 状态更新了，但组件没有重新渲染

可能原因：
1. useAppSelector 的引用比较问题
2. Redux 的不可变更新有问题

检查发现：
existing.quantity += 1 是直接修改，虽然 Redux Toolkit 允许，
但可能没有触发重新渲染

修复方案：
```javascript
addItem: (state, action) => {
  const existing = state.items.find(i => i.id === action.payload.id)
  if (existing) {
    existing.quantity += 1  // RTK 的 immer 会处理
  } else {
    state.items.push({ ...action.payload, quantity: 1 })
  }
}
```

实际上 Redux Toolkit 应该能处理，问题可能在组件：

```javascript
// 错误：返回了同一个数组引用
const items = useAppSelector(state => state.cart.items)

// 正确：确保重新计算
const items = useAppSelector(state => state.cart.items)
const total = useMemo(() => 
  items.reduce((sum, i) => sum + i.quantity, 0), 
  [items]
)
```

或者检查是否有其他地方修改了 state.items 而没有通过 reducer。
```

### 案例 2：内存泄漏问题

**问题描述：**

```
Bug：页面切换后，控制台还在打印日志，内存占用持续增长
复现：在页面 A 和页面 B 之间来回切换几次
技术栈：React + useEffect + WebSocket
```

**让 Claude Code 分析：**

```
页面切换后还有日志输出，怀疑是内存泄漏。

页面 A 的代码：
```javascript
function PageA() {
  useEffect(() => {
    const ws = new WebSocket('ws://example.com')
    ws.onmessage = (e) => {
      console.log('Received:', e.data)
    }
  }, [])
  
  return <div>Page A</div>
}
```

请分析问题并给出修复方案。
```

**Claude Code 分析：**

```
问题定位：
WebSocket 连接在组件卸载时没有关闭

修复方案：
```javascript
function PageA() {
  useEffect(() => {
    const ws = new WebSocket('ws://example.com')
    ws.onmessage = (e) => {
      console.log('Received:', e.data)
    }
    
    // 清理函数：组件卸载时执行
    return () => {
      ws.close()
      console.log('WebSocket closed')
    }
  }, [])
  
  return <div>Page A</div>
}
```

通用规则：
所有在 useEffect 中创建的副作用（订阅、定时器、事件监听），
都要在清理函数中清除。

检查清单：
- WebSocket → ws.close()
- setInterval → clearInterval
- setTimeout → clearTimeout
- addEventListener → removeEventListener
- 订阅 → 取消订阅
```

### 案例 3：竞态条件问题

**问题描述：**

```
Bug：搜索结果偶尔显示错误的结果
复现：快速输入 "abc"，有时显示 "ab" 的搜索结果
技术栈：React + 异步搜索 API
```

**让 Claude Code 分析：**

```
搜索结果错乱，快速输入时会显示旧关键词的结果。

搜索组件：
```javascript
function Search() {
  const [query, setQuery] = useState('')
  const [results, setResults] = useState([])

  useEffect(() => {
    fetch(`/api/search?q=${query}`)
      .then(res => res.json())
      .then(data => setResults(data))
  }, [query])

  return (
    <input onChange={e => setQuery(e.target.value)} value={query} />
    <ul>{results.map(r => <li>{r}</li>)}</ul>
  )
}
```

请分析问题并修复。
```

**Claude Code 分析：**

```
问题定位：
竞态条件（Race Condition）

输入 "abc" 的过程：
1. 输入 "a" → 发起请求 1
2. 输入 "ab" → 发起请求 2
3. 输入 "abc" → 发起请求 3

如果请求 2 比请求 3 慢，会先显示请求 3 的结果，
然后请求 2 返回，覆盖成 "ab" 的结果。

修复方案 1：取消旧请求
```javascript
useEffect(() => {
  const controller = new AbortController()
  
  fetch(`/api/search?q=${query}`, { signal: controller.signal })
    .then(res => res.json())
    .then(data => setResults(data))
    .catch(err => {
      if (err.name !== 'AbortError') throw err
    })
  
  return () => controller.abort()
}, [query])
```

修复方案 2：只处理最新请求
```javascript
useEffect(() => {
  let cancelled = false
  
  fetch(`/api/search?q=${query}`)
    .then(res => res.json())
    .then(data => {
      if (!cancelled) setResults(data)
    })
  
  return () => { cancelled = true }
}, [query])
```

修复方案 3：使用 SWR 或 React Query（推荐）
```javascript
import useSWR from 'swr'

function Search() {
  const [query, setQuery] = useState('')
  const { data: results } = useSWR(`/api/search?q=${query}`, fetcher)
  
  // SWR 自动处理竞态条件
}
```
```

## 🔧 Bug 修复工作流

### 标准流程

```
第一步：描述问题
→ Bug：[具体描述]，复现步骤：[步骤]

第二步：分析原因
→ 可能原因是什么？按可能性排序。

第三步：定位问题
→ 帮我加日志/断点定位到具体位置。

第四步：修复代码
→ 问题定位到了，帮我修复。

第五步：验证修复
→ 帮我写测试用例验证修复有效。

第六步：预防复发
→ 怎么避免类似问题？加什么检查？
```

### 常见 Bug 类型速查

| Bug 类型 | 典型表现 | 定位方法 |
|---------|---------|---------|
| 空值引用 | Cannot read property X of undefined | 检查数据流，加可选链 |
| 异步竞态 | 结果偶现错乱 | AbortController 或标记位 |
| 内存泄漏 | 内存持续增长 | 检查 useEffect 清理函数 |
| 状态不同步 | UI 和数据不一致 | 检查状态更新和订阅 |
| 类型错误 | TypeError | TypeScript 或运行时检查 |
| 边界问题 | 极端情况崩溃 | 边界值测试 |
