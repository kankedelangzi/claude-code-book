# 4.3 测试用例生成

> 手动测试太累？让 Claude Code 自动生成测试。

## 🎯 场景描述

你写了一个工具函数，但不确定边界情况有没有覆盖：

```javascript
function parsePrice(text) {
  // 去掉货币符号，转成数字
  return parseFloat(text.replace(/[^0-9.]/g, ''))
}
```

你希望测试：
- 普通数字：`"$99.99"` → `99.99`
- 只有数字：`"199"` → `199`
- 异常输入：`"abc"` → ？
- 空字符串：`""` → ？
- 负数：`"-$50"` → `？

手动写测试用例太麻烦。

## ✨ Claude Code 做法

**直接让它生成测试：**

```
帮我为这个函数写测试用例：

```javascript
function parsePrice(text) {
  return parseFloat(text.replace(/[^0-9.]/g, ''))
}
```

需要覆盖：正常输入、空值、非法输入、边界情况
```

**生成 Jest 测试：**

```javascript
describe('parsePrice', () => {
  it('处理普通价格格式', () => {
    expect(parsePrice('$99.99')).toBe(99.99)
  })

  it('处理纯数字', () => {
    expect(parsePrice('199')).toBe(199)
  })

  it('处理空字符串', () => {
    expect(parsePrice('')).toBeNaN()
  })

  it('处理非法输入', () => {
    expect(parsePrice('abc')).toBeNaN()
  })
})
```

**让测试驱动修复：**

```
这个函数的边界情况还没处理，帮我：
1. 列出所有可能的边界情况
2. 写测试用例覆盖它们
3. 如果需要修改函数实现，我来确认
```
