# 3.2 性能优化案例

> 让代码跑得更快，Claude Code 帮你找出瓶颈。

## 🎯 场景描述

代码能跑，但慢得离谱：

- 页面加载要 5 秒
- 数据库查询要 2 秒
- 批量处理要跑一整夜

**Claude Code 可以帮你分析性能瓶颈，给出优化方案。**

## ✨ 经典优化场景

### 场景 1：数据库 N+1 查询

**问题代码：**

```javascript
// 取出所有用户，然后逐一查他们的订单
function getAllUsersWithOrders() {
  const users = db.query('SELECT * FROM users');
  return users.map(user => ({
    ...user,
    orders: db.query('SELECT * FROM orders WHERE user_id = ?', user.id)
  }));
}
```

100 个用户 = 101 次数据库查询。

**优化指令：**

```
把这个改成一条 SQL，用 JOIN 一次性查出用户和订单。
输出优化后的 SQL 和对应的 JavaScript 代码。
```

**优化后：**

```javascript
function getAllUsersWithOrders() {
  const rows = db.query(`
    SELECT u.*, o.id as order_id, o.amount, o.created_at
    FROM users u
    LEFT JOIN orders o ON u.id = o.user_id
    ORDER BY u.id
  `);

  // 按用户分组
  const userMap = new Map();
  for (const row of rows) {
    if (!userMap.has(row.id)) {
      userMap.set(row.id, { ...row, orders: [] });
    }
    if (row.order_id) {
      userMap.get(row.id).orders.push({
        id: row.order_id,
        amount: row.amount,
        created_at: row.created_at
      });
    }
  }

  return Array.from(userMap.values());
}
```

**100 个用户 = 1 次数据库查询。**

### 场景 2：循环中的异步调用

**问题代码：**

```javascript
async function processOrders(orderIds) {
  const results = [];
  for (const id of orderIds) {
    const result = await processOrder(id);  // 逐个等待
    results.push(result);
  }
  return results;
}
```

100 个订单 = 串行处理，每个 100ms = 10 秒。

**优化指令：**

```
把这些串行的 await 改成并行处理。
用 Promise.all 批量处理，但限制并发数不超过 10，避免压垮服务器。
```

**优化后：**

```javascript
async function processOrders(orderIds, concurrency = 10) {
  const results = [];
  for (let i = 0; i < orderIds.length; i += concurrency) {
    const batch = orderIds.slice(i, i + concurrency);
    const batchResults = await Promise.all(batch.map(id => processOrder(id)));
    results.push(...batchResults);
  }
  return results;
}
```

**100 个订单 = 并行处理，约 1 秒完成（10 批 × 100ms）。**

### 场景 3：大文件处理内存爆炸

**问题代码：**

```javascript
function processLargeCSV(filepath) {
  const content = fs.readFileSync(filepath, 'utf-8');
  const lines = content.split('\n');
  const results = [];

  for (const line of lines) {
    const parsed = parseCSVLine(line);
    results.push(transformData(parsed));
  }

  return results;  // 全部加载到内存
}
```

GB 级别的 CSV 文件直接 OOM。

**优化指令：**

```
改成流式处理，一行一行读，一行一行处理，不一次性加载全部内容。
处理结果也分批写入，不要全部存内存。
```

**优化后：**

```javascript
const readline = require('readline');
const { createReadStream, createWriteStream } = require('fs');
const { Transform } = require('stream');

function processLargeCSV(inputPath, outputPath) {
  const rl = readline.createInterface({
    input: createReadStream(inputPath),
    crlfDelay: Infinity
  });

  const writeStream = createWriteStream(outputPath);
  let count = 0;

  rl.on('line', (line) => {
    const parsed = parseCSVLine(line);
    const transformed = transformData(parsed);
    writeStream.write(JSON.stringify(transformed) + '\n');
    count++;
    if (count % 10000 === 0) {
      console.log(`Processed ${count} lines...`);
    }
  });

  rl.on('close', () => {
    console.log(`Done! Total: ${count} lines`);
    writeStream.end();
  });
}
```

**GB 级别文件轻松处理，内存占用稳定。**

### 场景 4：重复计算

**问题代码：**

```javascript
function getReport(userId, year) {
  const user = getUser(userId);
  const orders = getOrdersByYear(userId, year);
  const stats = calculateStats(orders);
  const report = generateReport(user, stats);

  // 后面又用到了相同数据
  const email = generateEmail(user.name, stats.total);
  sendEmail(user.email, email);

  return report;
}
```

每个请求都重复查数据库、重复计算。

**优化指令：**

```
提取公共数据获取逻辑，用 Cache 缓存计算结果。
缓存有效期设为 5 分钟。
```

**优化后：**

```javascript
const cache = new Map();
const CACHE_TTL = 5 * 60 * 1000;

async function getCached(key, fn) {
  const cached = cache.get(key);
  if (cached && Date.now() - cached.time < CACHE_TTL) {
    return cached.value;
  }
  const value = await fn();
  cache.set(key, { value, time: Date.now() });
  return value;
}

async function getReport(userId, year) {
  const user = await getCached(`user:${userId}`, () => getUser(userId));
  const orders = await getCached(`orders:${userId}:${year}`, () => getOrdersByYear(userId, year));
  const stats = await getCached(`stats:${userId}:${year}`, () => calculateStats(orders));

  const report = generateReport(user, stats);
  const email = generateEmail(user.name, stats.total);
  await sendEmail(user.email, email);

  return report;
}
```

**同一人同一年，多次请求只查一次数据库。**

## 📊 性能优化检查清单

| 检查项 | 工具 |
|--------|------|
| 数据库查询是否过多 | 开启 SQL 日志 |
| 是否有重复计算 | 添加计时日志 |
| 是否有不必要的循环 | 代码审查 |
| 内存是否持续增长 | 监控进程内存 |
| 是否有同步阻塞 | Node.js profiler |

## 📝 小结

性能优化的关键：

1. **先测量，再优化** — 不确定瓶颈在哪就先打日志/ profiler
2. **优化有代价** — 缓存复杂化代码，并行增加复杂度
3. **选择高回报的改** — 优化热点代码，不值得优化冷代码
4. **改完要验证** — 用相同的测试用例，对比优化前后的性能

**Claude Code 可以帮你写优化代码，但瓶颈在哪，需要你提供上下文。**
