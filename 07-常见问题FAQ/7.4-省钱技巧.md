# 7.4 省钱技巧

> Claude Code 按用量计费。不多花冤枉钱，也是一种能力。

## 🎯 核心思路

省钱不是少用，而是**用得聪明**。

```
省钱 = 减少无效 token + 善用本地能力 + 合理规划对话
```

## 💡 最重要的原则

### 能在本地搞定的，别让它处理

Claude Code 每次对话都要发送上下文给 API。上下文越大，花费越高。

```
# 浪费钱 ❌
让 Claude Code 帮你改一个变量名（整个文件都当上下文发出去）

# 省钱 ✅
编辑器 Cmd+D 直接改，0 元
```

### 每次对话只做一件事

```
# 浪费钱 ❌（一次对话 5 个任务）
"帮我创建用户表、写 API、写前端页面、写测试、写文档"

# 省钱 ✅（5 次短对话）
对话 1：创建用户表
对话 2：写 API
对话 3：写前端页面
对话 4：写测试
对话 5：写文档
```

每次对话结束后，上下文清零。新对话 = 新计费起点。

## 📝 具体省钱技巧

### 技巧 1：用 /compact 压缩上下文

对话越长，每次发送的内容越多，费用越高。

Claude Code 支持 `/compact` 命令，压缩对话历史：

```
# 当对话变长时
/compact

# 或附带指令
/compact 保留关键的技术决策和代码结构
```

压缩后，之前的对话会被摘要替代，后续发送的 token 大幅减少。

**建议：对话超过 10 轮，就 compact 一次。**

### 技巧 2：精准指定文件

```
# 浪费钱 ❌
"帮我修复那个 bug"（它会扫描整个项目）

# 省钱 ✅
"修复 src/api/users.ts 第 45 行的 bug"
```

告诉它精确位置，它就不用到处搜索。

### 技巧 3：善用 .claudeignore

项目里有些文件完全没必要让 Claude Code 知道：

**.claudeignore**

```
# 忽略大型生成文件
dist/
build/
*.min.js
*.min.css

# 忽略媒体文件
images/
videos/
*.mp3
*.mp4

# 忽略依赖目录
node_modules/
vendor/

# 忽略日志
logs/
*.log

# 忽略大型数据文件
data/*.csv
data/*.json
```

文件越少，上下文越小，花费越少。

### 技巧 4：使用自定义指令减少重复

每次都要说"用 TypeScript""代码加注释""用 Tailwind"？

写进 `~/.claude/instructions.md`：

```markdown
# 自定义指令
- 使用 TypeScript
- 注释用中文
- 样式用 Tailwind CSS
- 优先 async/await
```

省下的不仅是钱，还有你的耐心。

### 技巧 5：先让它给方案，再让它写代码

```
# 浪费钱 ❌
"帮我重构这个模块"
（它可能写了一堆你不需要的代码）

# 省钱 ✅
"帮我分析这个模块的优化方向，只列方案不写代码"
（看完方案后）
"按方案 2 实现"
```

先确认方向，再花 token 写代码。方向错了，代码白写。

### 技巧 6：小改动自己来

| 操作 | 用 Claude Code？ | 原因 |
|------|-----------------|------|
| 改一个变量名 | ❌ | 编辑器全局替换更快 |
| 加一行注释 | ❌ | 手打最快 |
| 改一个 CSS 颜色 | ❌ | 手改最准 |
| 创建新组件 | ✅ | 值得花这个钱 |
| 调复杂 Bug | ✅ | 它比你快 |
| 写测试用例 | ✅ | 省时间也省钱 |

**3 分钟能搞定的，别花 token。**

### 技巧 7：合理使用模型

不同任务用不同级别的模型：

```
简单格式化、文件操作
    → 用较轻量的模型（如果可选）

复杂架构设计、疑难 Bug
    → 用最强模型

日常开发
    → 默认模型就好
```

## 📊 理解计费方式

Claude Code 的费用主要来自两方面：

```
1. 输入 token（你发送给 API 的内容）
   - 包含：你的指令 + 代码文件 + 对话历史 + 系统提示
   
2. 输出 token（API 返回的内容）
   - 包含：生成的代码 + 解释文字
```

**输入 token 通常比输出贵**（或者至少同样贵），所以减少输入量 = 直接省钱。

### 什么最吃 token？

```
大文件读取     ← 最大的 token 消耗
长对话历史     ← 越聊越贵
多文件搜索     ← 搜索结果都要发送回去
大型项目结构   ← 文件树本身就很多 token
```

### 什么不花 token？

```
终端命令执行     ← 执行本身不花 token（但输出会计入上下文）
文件读写         ← 读文件的内容会计入，写文件不额外花
```

## 🧩 一个估算公式

```
单次对话成本 ≈ (系统提示 + 你的指令 + 文件内容 + 对话历史) × 输入单价
             + (生成内容) × 输出单价
```

所以省钱的关键就是：
1. **减少每次对话传入的文件大小**
2. **压缩对话历史**
3. **指令简洁精准**

## ⚡ 实用省钱清单

每天开始工作前：

- [ ] 检查 `.claudeignore` 是否覆盖了不需要的文件
- [ ] 确认 `instructions.md` 里有常用偏好（省得每次重复说）

每次对话前：

- [ ] 这个任务我自己 3 分钟能搞定吗？能就自己做
- [ ] 文件路径我明确吗？明确就精准指定

对话过程中：

- [ ] 对话超过 10 轮了吗？超过就 `/compact`
- [ ] 它在搜索我不需要的东西吗？及时叫停

对话结束后：

- [ ] 涉及关键决策的内容记录下来了吗？（下次就不用重新讨论）

## 🎓 学到了什么

1. **省钱的核心是减少无效 token** — 精准指令 + 合理范围 + 及时压缩
2. **小改动自己做** — 别拿大炮打蚊子
3. **善用配置** — `.claudeignore` 和 `instructions.md` 是省钱利器
4. **先方案后代码** — 方向确认了再花 token

## ⚠️ 注意

- 省钱的前提是**效率不降低**，如果为了省钱让开发变慢，反而亏了
- Claude Code 的定价可能调整，关注官方公告
- 记住：**时间 > token 费用** — 别为了省几毛钱浪费半小时
