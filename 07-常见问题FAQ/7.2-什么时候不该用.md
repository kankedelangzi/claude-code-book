# 7.2 什么时候不该用？

> 知道什么时候不用，比知道什么时候用更重要。

## 🎯 一句话原则

**能用键盘三分钟搞定的，别打开 Claude Code。**

## ❌ 不该用的场景

### 1. 改一个变量名

```
# 你
把 config.username 改成 config.accountName

# 正确做法
编辑器全局替换，10 秒搞定

# 用 Claude Code
等它读文件、分析、修改、确认... 30 秒
```

### 2. 一行代码的事

```javascript
// 你想加一行 console.log
console.log('debug:', data);
```

别让 Claude Code 帮你写一行 console.log。真的。

### 3. 你还不知道要做什么

```
帮我把这个项目改得更好一点
```

这种模糊的需求给谁都头疼。先想清楚，再动手。

Claude Code 需要明确的输入，否则它会给你明确的废话。

### 4. 涉及敏感信息的操作

```
帮我生成一个数据库迁移脚本，用户表里加个字段
```

如果这个脚本会直接跑在生产数据库上——**停**。

涉及以下内容的操作，先自己确认，再决定是否让 Claude Code 参与：
- 数据库 schema 变更
- 删除操作（文件、记录、分支）
- 生产环境部署脚本
- 权限配置
- 密钥和证书

### 5. 需要深度理解的业务逻辑

```
帮我把这个支付回调的处理逻辑重写一下
```

支付逻辑、权限控制、合规代码——这些需要你完全理解每一行在干什么。

Claude Code 可以帮你**写**，但不该让它帮你**设计**这种逻辑。

### 6. 代码 Review 是你自己做的

```
帮我 review 一下这个 PR
```

Claude Code 可以帮你找出明显的代码问题，但：
- 它不知道团队的编码规范
- 它不了解业务上下文
- 它不会说"这段代码上次上线出了问题"

**用 Claude Code 做 pre-review（初筛），自己做最终判断。**

### 7. 别在紧急情况下第一次用

```
（线上报警 500）
快快快帮我修一下！
```

生产环境出问题时，不是学新工具的好时机。

确保你平时用 Claude Code 已经熟悉了，关键时刻才能信任它。

## ⚠️ 容易踩的坑

### 坑 1：无脑复制粘贴

Claude Code 给的代码，你不看就粘贴运行？

```
它给的 SQL 里可能有 DROP TABLE
它给的正则可能不匹配你的数据
它给的配置可能和你的环境冲突
```

**每一段代码都要看懂再执行。**

### 坑 2：越改越大

```
帮我改一下这个函数
   ↓
这个函数依赖的那个也要改
   ↓
那个的接口也需要调整
   ↓
算了，整个模块重构吧
```

原来的小改动变成了大工程，然后越改越多，越改越乱。

**给 Claude Code 设定明确的范围，一次只做一件事。**

### 坑 3：失去手感

连续用 Claude Code 写了一个月代码，发现自己连基本的语法都要想半天。

这不是 Claude Code 的错，但你要有意识地保持手动编码的习惯。

**建议：简单代码自己写，复杂代码用它辅助。**

### 坑 4：过度依赖搜索

```
帮我看一下这个 npm 包怎么用
帮我看一下这个 API 文档
帮我看一下这个报错什么意思
```

偶尔搜索没问题，但如果每一行代码都要它帮你查——你在用它，还是它在替你思考？

## ✅ 该用的场景（对照一下）

| 场景 | 用还是不用 |
|------|-----------|
| 创建新组件/页面 | ✅ 用 |
| 修改现有功能的逻辑 | ⚠️ 用，但要自己 review |
| 全局搜索替换 | ✅ 用，比编辑器更智能 |
| 写测试用例 | ✅ 用 |
| 调 Bug | ✅ 用，但最终修复你自己确认 |
| 代码 Review | ⚠️ 辅助用，不能替代人工 |
| 写一行 console.log | ❌ 别用了 |
| 修个拼写错误 | ❌ 别用了 |
| 改个配置项 | ❌ 大多数情况别用了 |
| 生产环境操作 | ❌ 不用，自己来 |
| 敏感数据操作 | ❌ 不用，自己来 |

## 📝 一个简单的判断流程

```
要做的任务
    │
    ├─ 明确目标吗？
    │   ├─ 不明确 → 先自己想清楚，不用
    │   └─ 明确 ↓
    │
    ├─ 自己做 < 3 分钟？
    │   ├─ 是 → 不用，自己做
    │   └─ 否 ↓
    │
    ├─ 涉及生产/敏感操作？
    │   ├─ 是 → 不用，自己来
    │   └─ 否 ↓
    │
    └─ 用 Claude Code！
```

## 🎓 学到了什么

1. **Claude Code 是效率工具，不是万能钥匙** — 合理使用才能发挥最大价值
2. **简单的事情自己做** — 启动成本高于执行成本时，别用它
3. **保持独立思考能力** — 它的建议要过脑子，不能无脑采纳
4. **了解边界才能信任** — 知道它什么时候会出错，才能放心让它在擅长的地方工作
