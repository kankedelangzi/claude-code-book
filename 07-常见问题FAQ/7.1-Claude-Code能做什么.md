# 7.1 Claude Code 能做什么？

> 别问"它行不行"，问"怎么让它干"。大多数时候，答案是：能。

## 🎯 一句话回答

Claude Code 是一个住在终端里的编程搭档——你说需求，它写代码、改代码、调 Bug、跑测试，还能帮你读文档和搜索方案。

## 💡 它擅长的领域

### 1. 写代码

```
帮我在 Next.js 项目里创建一个用户登录页面，
用 TypeScript，包含表单验证和错误提示
```

它从零开始写，连组件文件都帮你建好。

### 2. 改代码

```
把这个函数从 class 写法改成函数式写法
```

```
给这个 API 接口加上 JWT 鉴权中间件
```

不用自己翻来覆去找要改的地方。

### 3. 读代码

```
解释一下 src/utils/auth.ts 这个文件在做什么
```

比你自己一行行看快得多，还会指出潜在问题。

### 4. 调 Bug

```
这个接口返回 500，日志里有 "Cannot read property 'map' of undefined"
帮我定位问题
```

它会读代码、分析调用链、找到问题根源，甚至直接给出修复方案。

### 5. 跑命令

```
帮我创建一个 git 分支 feature/user-auth 并切换过去
```

```
运行测试，把失败的用例列出来
```

终端能干的事它都能干。

### 6. 搜索和调研

```
帮我查一下 Prisma 和 Drizzle ORM 的区别
```

```
这个报错是什么意思：ERESOLVE unable to resolve dependency tree
```

自带搜索能力，不用切浏览器。

### 7. 写测试

```
给 src/services/userService.ts 写单元测试，
用 Jest，覆盖正常和异常情况
```

### 8. 写文档

```
帮我写一个 API 文档，覆盖所有路由
```

## 🔧 它能操作什么

| 能力 | 示例 |
|------|------|
| 读写文件 | 创建、编辑、删除项目文件 |
| 执行命令 | npm、git、docker 等终端命令 |
| 搜索代码 | 全项目 grep，精准定位 |
| 浏览网页 | 查文档、找解决方案 |
| 操作 Git | 提交、分支、合并、解决冲突 |

## ❌ 它做不好的事

**诚实说，有些场景别指望它：**

1. **完全取代你的判断** — 它写的代码你要 review，不要无脑采纳
2. **理解你没有说清的需求** — 指令模糊，结果就模糊
3. **处理超大型单体项目** — 上下文窗口有限，几十万行的项目会吃力
4. **实时协作** — 它不是实时结对编程，是异步对话
5. **闭眼提交到生产环境** — 重要代码必须自己测试

## 🧩 实际能力边界

### 能做的 ✅

```
创建一个 React 组件         → 没问题
重构一个 200 行的函数       → 没问题
修复一个报错                → 大部分没问题
写一个 REST API            → 没问题
解释一段陌生代码            → 没问题
```

### 勉强能做的 ⚠️

```
理解整个微服务架构           → 需要你逐步引导
从零设计数据库              → 建议你自己设计好结构再让它实现
修改不熟悉的框架代码         → 可能需要你补充上下文
处理复杂的 Git 冲突         → 简单冲突可以，复杂的你要介入
```

### 做不了的 ❌

```
替你做技术决策              → 它能给建议，但拍板得你来
保证代码 100% 正确          → 永远要 review
理解项目的历史背景           → 它只看代码，不看人情世故
处理硬件问题                → 软件的事它能帮，拔网线的事它不行
```

## 📝 怎么判断该不该用它？

一个简单的决策标准：

```
这个任务需要 < 10 分钟自己写？
    → 不需要 Claude Code，自己敲更快

这个任务需要 10 分钟 ~ 2 小时？
    → 用 Claude Code，效率能翻 3-5 倍

这个任务需要 > 2 小时或者我完全不会？
    → 用 Claude Code，但要多轮对话，逐步推进
```

## 🎓 学到了什么

1. **Claude Code 是编程搭档，不是魔法** — 给它清晰的指令才有好结果
2. **它最擅长的是"有明确目标的编码任务"** — 越具体，效果越好
3. **了解边界比知道能力更重要** — 知道它做不好什么，才能用好它

## ⚠️ 注意

- Claude Code 的能力在不断更新，保持关注官方文档
- 不同模型版本能力有差异，遇到问题先确认版本
- 它是你的工具，不是你的替代品
