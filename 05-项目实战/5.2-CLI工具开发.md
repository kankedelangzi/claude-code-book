# 5.2 CLI 工具开发

> 从零做一个有用的命令行工具，体会完整项目开发流程。

## 🎯 场景描述

你想做一个命令行工具：

- 批量重命名文件
- 快速生成代码模板
- 自动化日常操作
- 内部工具分享给同事

**Claude Code 可以帮你从需求分析到完整实现，做出一个真正能用的 CLI 工具。**

## ✨ 案例：做一个 "文件整理器"

### 需求明确

你想做一个整理文件的 CLI：

```
用法: organize [命令] [选项]

命令:
  by-ext    按文件扩展名整理
  by-date   按修改日期整理
  by-size   按文件大小整理
  by-name   按文件名关键词整理

选项:
  -p, --path <路径>    目标目录（默认当前目录）
  -d, --dry-run        预览模式，不实际移动文件
  -v, --verbose        显示详细日志
  -h, --help          显示帮助

示例:
  organize by-ext -p ~/Downloads
  organize by-name -p ~/Documents -d "截图,照片"
```

organize-cli/

要求：

program.parse();

### 第二步：实现核心功能

**对话：**

```
实现 by-ext.js 命令：

功能：
1. 扫描目标目录所有文件
2. 按扩展名分组（.jpg, .png → images/，.pdf, .doc → documents/）
3. 创建对应的分类目录
4. 移动或复制文件到分类目录
5. 支持 --dry-run 模式，只打印移动计划，不实际执行

扩展名映射：
images: jpg, jpeg, png, gif, webp, svg, bmp
documents: pdf, doc, docx, txt, md, rtf
videos: mp4, avi, mov, mkv, webm
audio: mp3, wav, flac, aac, ogg
archives: zip, rar, 7z, tar, gz
code: js, ts, py, java, go, rs, html, css
other: 其他所有

输出格式：
[INFO] 扫描: /path/to/dir (共 X 个文件)
[INFO] 分组: images(X) documents(Y) ...
[DRY-RUN] would move: file.jpg → images/file.jpg
[SUCCESS] 完成！整理了 X 个文件
```

**Claude Code 实现：**

```javascript
// src/commands/by-ext.js
import fs from 'fs';
import path from 'path';
import { logger } from '../utils/logger.js';

const EXT_GROUPS = {
  images: ['jpg', 'jpeg', 'png', 'gif', 'webp', 'svg', 'bmp'],
  documents: ['pdf', 'doc', 'docx', 'txt', 'md', 'rtf'],
  videos: ['mp4', 'avi', 'mov', 'mkv', 'webm'],
  audio: ['mp3', 'wav', 'flac', 'aac', 'ogg'],
  archives: ['zip', 'rar', '7z', 'tar', 'gz'],
  code: ['js', 'ts', 'py', 'java', 'go', 'rs', 'html', 'css'],
};

function getCategory(ext) {
  ext = ext.toLowerCase();
  for (const [category, exts] of Object.entries(EXT_GROUPS)) {
    if (exts.includes(ext)) return category;
  }
  return 'other';
}

export async function byExt(options) {
  const { path: targetPath, dryRun, verbose } = options;
  const logger = createLogger(verbose);

  logger.info(`扫描: ${targetPath}`);

  // 1. 获取所有文件
  const files = fs.readdirSync(targetPath).filter(f => {
    return fs.statSync(path.join(targetPath, f)).isFile();
  });

  logger.info(`共 ${files.length} 个文件`);

  // 2. 按扩展名分组
  const groups = {};
  for (const file of files) {
    const ext = path.extname(file).slice(1) || 'no-ext';
    const category = getCategory(ext);
    groups[category] = groups[category] || [];
    groups[category].push(file);
  }

  // 3. 输出分组统计
  for (const [cat, catFiles] of Object.entries(groups)) {
    logger.info(`  ${cat}: ${catFiles.length} 个`);
  }

  // 4. 执行移动（或预览）
  let movedCount = 0;
  for (const [category, catFiles] of Object.entries(groups)) {
    if (catFiles.length === 0) continue;

    const destDir = path.join(targetPath, category);
    
    if (!dryRun && !fs.existsSync(destDir)) {
      fs.mkdirSync(destDir, { recursive: true });
    }

    for (const file of catFiles) {
      const src = path.join(targetPath, file);
      const dest = path.join(destDir, file);

      if (dryRun) {
        logger.dryRun(`would move: ${file} → ${category}/${file}`);
      } else {
        fs.renameSync(src, dest);
        logger.success(`moved: ${file} → ${category}/`);
      }
      movedCount++;
    }
  }

  logger.success(`完成！整理了 ${movedCount} 个文件`);
}
```

### 第三步：添加日志工具

**对话：**

```
写一个 logger.js 工具模块：

功能：
- 带颜色的日志输出
- 支持不同级别：info, success, warn, error
- 支持 --dry-run 模式的特殊标记 [DRY-RUN]
- 支持 --verbose 的详细输出

颜色方案：
- info: 蓝色
- success: 绿色
- warn: 黄色
- error: 红色
- dry-run: 灰色
```

**Claude Code 实现：**

```javascript
// src/utils/logger.js
import chalk from 'chalk';

export function createLogger(verbose = false) {
  return {
    info: (msg) => console.log(chalk.blue('[INFO]'), msg),
    success: (msg) => console.log(chalk.green('[SUCCESS]'), msg),
    warn: (msg) => console.log(chalk.yellow('[WARN]'), msg),
    error: (msg) => console.log(chalk.red('[ERROR]'), msg),
    dryRun: (msg) => console.log(chalk.gray('[DRY-RUN]'), msg),
    verbose: (...args) => verbose && console.log(...args),
  };
}
```

## 📝 小结

用 Claude Code 开发 CLI 工具的流程：

1. **先想清楚** — 写出用法示例，Claude 就能照着实现
2. **分模块** — 命令、工具函数分开，便于维护
3. **先跑通** — `--dry-run` 模式是调试神器
4. **再优化** — 颜色日志、配置支持这些"小事"最后加

**一个 CLI 工具的价值不在于代码量，而在于真的能省时间。**
