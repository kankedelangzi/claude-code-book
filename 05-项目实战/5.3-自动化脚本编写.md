# 5.3 自动化脚本编写

> 不写重复代码，让机器干脏活累活。

## 🎯 场景描述

日常开发中，总有些重复性工作：

- 每天定时备份数据库
- 批量压缩图片、转换格式
- 监控日志文件，发现错误就报警
- 定时清理临时文件
- 批量重命名、移动文件

**Claude Code 可以帮你快速写出这些脚本，一次编写，永久省心。**

## ✨ 案例一：日志监控报警脚本

### 需求明确

监控应用日志，发现 ERROR 级别日志就发送通知：

```
功能：
- 实时监控日志文件新增内容
- 匹配 ERROR/FATAL/EXCEPTION 关键字
- 发现错误时发送企业微信/钉钉/邮件通知
- 支持错误去重（同一错误 5 分钟内不重复报警）
- 记录报警历史

用法: node log-monitor.js <日志文件路径>
```

---

### 第一步：基础版本

**对话：**

```
帮我写一个 Node.js 脚本，实时监控日志文件：

要求：
1. 使用 fs.watchFile 或 tail 方式监控文件变化
2. 读取新增的行内容
3. 用正则匹配 ERROR、FATAL、EXCEPTION
4. 打印到控制台：时间 + 错误内容
5. 支持优雅退出（Ctrl+C 时清理资源）
```

**Claude Code 实现：**

```javascript
// log-monitor.js
import fs from 'fs';
import { createReadStream } from 'fs';
import { readFile } from 'fs/promises';
import path from 'path';

const LOG_FILE = process.argv[2];

if (!LOG_FILE) {
  console.error('用法: node log-monitor.js <日志文件路径>');
  process.exit(1);
}

// 错误匹配正则
const ERROR_PATTERNS = [
  /\bERROR\b/i,
  /\bFATAL\b/i,
  /\bEXCEPTION\b/i,
  /\bCRITICAL\b/i
];

let lastSize = 0;

// 初始化：获取当前文件大小
async function init() {
  try {
    const stats = await fs.promises.stat(LOG_FILE);
    lastSize = stats.size;
    console.log(`[${timestamp()}] 开始监控: ${LOG_FILE}`);
    console.log(`[${timestamp()}] 当前文件大小: ${lastSize} bytes`);
    console.log(`[${timestamp()}] 按 Ctrl+C 停止监控\n`);
  } catch (err) {
    console.error(`[${timestamp()}] 错误: 无法读取文件 ${LOG_FILE}`);
    console.error(err.message);
    process.exit(1);
  }
}

// 检查新增内容
async function checkNewContent() {
  try {
    const stats = await fs.promises.stat(LOG_FILE);
    const currentSize = stats.size;

    if (currentSize > lastSize) {
      // 读取新增部分
      const stream = createReadStream(LOG_FILE, {
        start: lastSize,
        end: currentSize - 1
      });

      let buffer = '';
      stream.on('data', chunk => {
        buffer += chunk;
      });

      stream.on('end', () => {
        const lines = buffer.split('\n').filter(line => line.trim());
        lines.forEach(line => checkLine(line));
        lastSize = currentSize;
      });
    }
  } catch (err) {
    console.error(`[${timestamp()}] 读取错误:`, err.message);
  }
}

// 检查单行内容
function checkLine(line) {
  for (const pattern of ERROR_PATTERNS) {
    if (pattern.test(line)) {
      console.log(`[${timestamp()}] ⚠️ 发现错误:`);
      console.log(`  ${line.trim()}`);
      console.log('');
      break;
    }
  }
}

// 时间戳
function timestamp() {
  return new Date().toLocaleString('zh-CN');
}

// 主循环
async function main() {
  await init();

  // 每秒检查一次
  const interval = setInterval(checkNewContent, 1000);

  // 优雅退出
  process.on('SIGINT', () => {
    console.log(`\n[${timestamp()}] 停止监控`);
    clearInterval(interval);
    process.exit(0);
  });
}

main();
```

### 第二步：添加报警去重

**对话：**

```
优化脚本，添加报警去重功能：

要求：
1. 用内存记录最近 5 分钟内报过的错误
2. 同一错误（内容相同或相似）不重复报警
3. 超过 5 分钟的错误从内存中清除
4. 添加 --dedup-time 参数控制去重时间（默认 5 分钟）
```

**Claude Code 实现：**

```javascript
// 添加去重逻辑
class ErrorDeduper {
  constructor(windowMinutes = 5) {
    this.windowMs = windowMinutes * 60 * 1000;
    this.recentErrors = new Map(); // errorHash -> timestamp
  }

  // 生成错误指纹（简化版）
  getFingerprint(line) {
    // 提取关键部分：去掉时间戳、行号等变化内容
    return line
      .replace(/\d{4}-\d{2}-\d{2}[\sT]\d{2}:\d{2}:\d{2}/, '')
      .replace(/:\d+\)?$/, '')
      .replace(/\s+/g, ' ')
      .trim()
      .slice(0, 100); // 限制长度
  }

  shouldAlert(line) {
    const fingerprint = this.getFingerprint(line);
    const now = Date.now();

    // 清理过期记录
    this.cleanup(now);

    if (this.recentErrors.has(fingerprint)) {
      const lastTime = this.recentErrors.get(fingerprint);
      if (now - lastTime < this.windowMs) {
        return false; // 重复错误，不报警
      }
    }

    // 记录新错误
    this.recentErrors.set(fingerprint, now);
    return true;
  }

  cleanup(now) {
    for (const [fp, time] of this.recentErrors) {
      if (now - time > this.windowMs) {
        this.recentErrors.delete(fp);
      }
    }
  }

  stats() {
    return {
      tracked: this.recentErrors.size,
      windowMinutes: this.windowMs / 60000
    };
  }
}

// 修改 checkLine 函数
const deduper = new ErrorDeduper(5);

function checkLine(line) {
  for (const pattern of ERROR_PATTERNS) {
    if (pattern.test(line)) {
      if (deduper.shouldAlert(line)) {
        console.log(`[${timestamp()}] ⚠️ 发现错误:`);
        console.log(`  ${line.trim()}`);
        console.log('');
      } else {
        console.log(`[${timestamp()}] 🔄 重复错误（已抑制）`);
      }
      break;
    }
  }
}
```

### 第三步：添加企业微信通知

**对话：**

```
添加企业微信机器人通知功能：

要求：
1. 通过环境变量读取 webhook URL
2. 发现错误时发送 POST 请求到企业微信
3. 消息格式包含：时间、错误内容、日志文件名
4. 添加 --webhook 参数覆盖环境变量
5. 如果 webhook 配置错误，降级为仅控制台输出
```

**Claude Code 实现：**

```javascript
// 企业微信通知模块
class WeComNotifier {
  constructor(webhookUrl) {
    this.webhookUrl = webhookUrl || process.env.WECOM_WEBHOOK;
  }

  isConfigured() {
    return !!this.webhookUrl;
  }

  async sendAlert(errorLine, logFile) {
    if (!this.isConfigured()) {
      return false;
    }

    const message = {
      msgtype: 'markdown',
      markdown: {
        content: `**⚠️ 日志错误告警**\n\n` +
          `> 时间：${new Date().toLocaleString('zh-CN')}\n` +
          `> 文件：\`${logFile}\`\n` +
          `> 错误：\`\`\`${errorLine.slice(0, 500)}\`\`\``
      }
    };

    try {
      const response = await fetch(this.webhookUrl, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(message)
      });

      if (!response.ok) {
        console.error(`[${timestamp()}] 发送通知失败: ${response.status}`);
        return false;
      }

      return true;
    } catch (err) {
      console.error(`[${timestamp()}] 通知发送错误:`, err.message);
      return false;
    }
  }
}

// 修改主逻辑
const notifier = new WeComNotifier(process.argv[3]);

async function checkLine(line) {
  for (const pattern of ERROR_PATTERNS) {
    if (pattern.test(line)) {
      if (deduper.shouldAlert(line)) {
        console.log(`[${timestamp()}] ⚠️ 发现错误:`);
        console.log(`  ${line.trim()}`);

        // 发送通知
        if (notifier.isConfigured()) {
          const sent = await notifier.sendAlert(line, LOG_FILE);
          if (sent) {
            console.log(`[${timestamp()}] 📤 通知已发送`);
          }
        }

        console.log('');
      }
      break;
    }
  }
}
```

## ✨ 案例二：定时清理脚本

### 需求

定期清理指定目录的临时文件：

```
功能：
- 递归扫描目录
- 删除超过 N 天的文件
- 删除匹配特定模式的文件（如 *.tmp, *.log）
- 支持白名单（不删除某些文件）
- 支持 --dry-run 预览
- 输出清理报告

用法: node cleanup.js <目录> [选项]
```

**Claude Code 生成：**

```javascript
// cleanup.js
import fs from 'fs/promises';
import path from 'path';

const CONFIG = {
  defaultMaxAge: 7, // 默认 7 天
  patterns: ['*.tmp', '*.log', '*.cache', 'Thumbs.db', '.DS_Store'],
  whitelist: ['.gitkeep', 'important.log']
};

async function cleanup(targetDir, options = {}) {
  const { maxAge = CONFIG.defaultMaxAge, dryRun = false, verbose = false } = options;
  const maxAgeMs = maxAge * 24 * 60 * 60 * 1000;
  const now = Date.now();

  const stats = {
    scanned: 0,
    deleted: 0,
    skipped: 0,
    errors: 0,
    freedBytes: 0
  };

  async function scan(dir) {
    const entries = await fs.readdir(dir, { withFileTypes: true });

    for (const entry of entries) {
      const fullPath = path.join(dir, entry.name);

      if (entry.isDirectory()) {
        await scan(fullPath);
      } else {
        stats.scanned++;
        await processFile(fullPath, entry.name);
      }
    }
  }

  async function processFile(filePath, fileName) {
    // 白名单检查
    if (CONFIG.whitelist.includes(fileName)) {
      stats.skipped++;
      if (verbose) console.log(`[SKIP] 白名单: ${filePath}`);
      return;
    }

    try {
      const stat = await fs.stat(filePath);
      const age = now - stat.mtime.getTime();
      const matchesPattern = CONFIG.patterns.some(p => 
        fileName.toLowerCase().endsWith(p.replace('*', '').toLowerCase())
      );

      const shouldDelete = age > maxAgeMs || matchesPattern;

      if (shouldDelete) {
        if (dryRun) {
          console.log(`[DRY-RUN] 将删除: ${filePath} (${formatBytes(stat.size)})`);
        } else {
          await fs.unlink(filePath);
          console.log(`[DELETED] ${filePath} (${formatBytes(stat.size)})`);
        }
        stats.deleted++;
        stats.freedBytes += stat.size;
      }
    } catch (err) {
      stats.errors++;
      console.error(`[ERROR] ${filePath}: ${err.message}`);
    }
  }

  console.log(`开始清理: ${targetDir}`);
  console.log(`规则: 超过 ${maxAge} 天的文件 或 匹配模式 [${CONFIG.patterns.join(', ')}]`);
  console.log(`模式: ${dryRun ? '预览' : '实际删除'}\n`);

  await scan(targetDir);

  console.log(`\n========== 清理报告 ==========`);
  console.log(`扫描文件: ${stats.scanned}`);
  console.log(`删除文件: ${stats.deleted}`);
  console.log(`跳过文件: ${stats.skipped}`);
  console.log(`错误: ${stats.errors}`);
  console.log(`释放空间: ${formatBytes(stats.freedBytes)}`);
}

function formatBytes(bytes) {
  if (bytes === 0) return '0 B';
  const k = 1024;
  const sizes = ['B', 'KB', 'MB', 'GB'];
  const i = Math.floor(Math.log(bytes) / Math.log(k));
  return parseFloat((bytes / Math.pow(k, i)).toFixed(2)) + ' ' + sizes[i];
}

// CLI
const [,, targetDir, ...args] = process.argv;
if (!targetDir) {
  console.log('用法: node cleanup.js <目录> [--days=N] [--dry-run] [--verbose]');
  process.exit(1);
}

const options = {
  maxAge: args.find(a => a.startsWith('--days='))?.split('=')[1] || 7,
  dryRun: args.includes('--dry-run'),
  verbose: args.includes('--verbose')
};

cleanup(targetDir, options);
```

## ✨ 案例三：批量图片处理脚本

### 需求

批量压缩、转换图片格式：

```javascript
// image-batch.js
import fs from 'fs/promises';
import path from 'path';
import { exec } from 'child_process';
import { promisify } from 'util';

const execAsync = promisify(exec);

// 需要安装 sharp: npm install sharp
import sharp from 'sharp';

async function processImages(inputDir, outputDir, options = {}) {
  const { 
    width = 1920,      // 最大宽度
    quality = 80,      // JPEG 质量
    format = 'jpeg',   // 输出格式
    suffix = '_optimized'
  } = options;

  await fs.mkdir(outputDir, { recursive: true });

  const files = await fs.readdir(inputDir);
  const imageFiles = files.filter(f => /\.(jpg|jpeg|png|webp)$/i.test(f));

  console.log(`找到 ${imageFiles.length} 张图片，开始处理...\n`);

  let totalOriginal = 0;
  let totalOutput = 0;

  for (const file of imageFiles) {
    const inputPath = path.join(inputDir, file);
    const ext = path.extname(file);
    const baseName = path.basename(file, ext);
    const outputFile = `${baseName}${suffix}.${format}`;
    const outputPath = path.join(outputDir, outputFile);

    try {
      const originalStat = await fs.stat(inputPath);
      totalOriginal += originalStat.size;

      await sharp(inputPath)
        .resize(width, null, { withoutEnlargement: true })
        .toFormat(format, { quality })
        .toFile(outputPath);

      const outputStat = await fs.stat(outputPath);
      totalOutput += outputStat.size;

      const ratio = ((1 - outputStat.size / originalStat.size) * 100).toFixed(1);
      console.log(`✓ ${file} → ${outputFile} (节省 ${ratio}%)`);

    } catch (err) {
      console.error(`✗ ${file}: ${err.message}`);
    }
  }

  const totalRatio = ((1 - totalOutput / totalOriginal) * 100).toFixed(1);
  console.log(`\n========== 处理完成 ==========`);
  console.log(`原大小: ${formatBytes(totalOriginal)}`);
  console.log(`处理后: ${formatBytes(totalOutput)}`);
  console.log(`节省: ${totalRatio}%`);
}

function formatBytes(bytes) {
  return (bytes / 1024 / 1024).toFixed(2) + ' MB';
}

// 使用
const [,, input, output] = process.argv;
if (!input || !output) {
  console.log('用法: node image-batch.js <输入目录> <输出目录>');
  process.exit(1);
}

processImages(input, output);
```

## 🔧 让脚本定时运行

### 使用 cron（Linux/Mac）

```bash
# 编辑 crontab
crontab -e

# 每 5 分钟检查一次日志
*/5 * * * * cd /path/to/scripts && node log-monitor.js /var/log/app.log >> /var/log/monitor.log 2>&1

# 每天凌晨 2 点清理临时文件
0 2 * * * node /path/to/scripts/cleanup.js /tmp --days=3

# 每周日压缩图片
0 3 * * 0 node /path/to/scripts/image-batch.js /path/to/raw /path/to/optimized
```

### 使用 PM2（Node.js 守护进程）

```bash
# 安装 PM2
npm install -g pm2

# 启动监控脚本
pm2 start log-monitor.js --name log-monitor -- /var/log/app.log

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

## 📝 小结

写自动化脚本的套路：

| 步骤 | 说明 |
|------|------|
| 1. 明确需求 | 先写"用法"，再写代码 |
| 2. 核心逻辑 | 用最简单的方式先跑通 |
| 3. 容错处理 | 文件不存在、网络失败怎么办 |
| 4. 日志输出 | 让用户知道脚本在干什么 |
| 5. 参数化 | 硬编码 → 命令行参数 → 配置文件 |

**好脚本的标准：**
- 能 `--dry-run` 预览
- 有清晰的日志输出
- 失败时给出明确错误
- 不依赖特定环境（或明确声明依赖）
