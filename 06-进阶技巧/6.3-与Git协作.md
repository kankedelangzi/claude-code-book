# 6.3 与 Git 协作

> 让 Claude Code 帮你管理 Git，写出规范的提交记录。

## 🎯 场景描述

Git 操作繁琐：
- 提交信息不知道怎么写
- 代码改了一堆，不知道分几次提交
- 冲突解决麻烦

Claude Code 可以帮你处理这些。

## 💡 核心思路

```
让 Claude Code 生成规范的 commit message，帮你管理代码提交
```

## 📝 实战案例一：生成 Commit Message

### 需求描述

```
帮我看看改了什么，写一个 commit message
```

### Claude Code 执行

```bash
# 查看改动
git diff --stat
```

```
 src/api/users.ts    |  12 +++++++++--
 src/services/auth.ts |  5 ++--
 tests/users.test.ts  |  8 +++++++
 3 files changed, 21 insertions(+), 4 deletions(-)
```

### 生成的 Commit Message

```
feat: 添加用户角色字段，优化认证逻辑

- User 接口增加 role 字段（admin/user/guest）
- authService 支持角色验证
- 新增用户角色相关测试用例
```

符合 Conventional Commits 规范：
- `feat:` 新功能
- `fix:` Bug 修复
- `docs:` 文档更新
- `refactor:` 重构
- `test:` 测试相关

## 📝 实战案例二：智能分批提交

### 需求描述

```
我改了很多文件，帮我按功能分批提交
```

### Claude Code 分析后

```
发现以下改动：

1. 用户认证相关（建议单独提交）
   - src/services/auth.ts
   - src/middleware/auth.ts
   
2. 用户 API 相关（建议单独提交）
   - src/api/users.ts
   - src/types/user.ts
   
3. 测试文件（可以和对应功能一起提交，或单独提交）
   - tests/auth.test.ts
   - tests/users.test.ts

请选择：
A. 按上面的分组分 3 次提交
B. 全部一次提交
C. 自定义分组
```

### 选择 A，Claude Code 执行

```bash
# 第一次提交
git add src/services/auth.ts src/middleware/auth.ts
git commit -m "feat(auth): 添加角色验证中间件"

# 第二次提交
git add src/api/users.ts src/types/user.ts
git commit -m "feat(users): 用户 API 支持角色字段"

# 第三次提交
git add tests/
git commit -m "test: 添加认证和用户角色测试用例"
```

## 📝 实战案例三：解决合并冲突

### 需求描述

```
合并 dev 分支到 main，有冲突，帮我解决
```

### Claude Code 执行

```bash
git merge dev
# 提示冲突：src/config.ts
```

**冲突内容：**
```typescript
<<<<<<< HEAD
const API_URL = 'https://api.example.com';
=======
const API_URL = 'https://dev-api.example.com';
>>>>>>> dev
```

**Claude Code 分析：**
```
冲突分析：
- HEAD（main）: 生产环境 API
- dev: 开发环境 API

建议：使用环境变量，不要硬编码

解决方案：
const API_URL = process.env.API_URL || 'https://api.example.com';
```

自动解决冲突并提交。

## 📝 实战案例四：生成分支描述

### 需求描述

```
帮我创建一个新分支，描述清楚要做什么
```

### Claude Code

```bash
# 创建分支
git checkout -b feature/user-dashboard

# 创建分支说明文件
echo "# Feature: User Dashboard

## 目标
- 用户个人主页
- 展示用户活动统计
- 支持设置修改

## 相关 Issue
- #123 用户主页设计
- #124 统计图表组件

## 进度
- [ ] 页面布局
- [ ] 数据接口
- [ ] 前端组件
" > .branch-notes.md

git add .branch-notes.md
git commit -m "docs: 添加分支开发说明"
```

## ⚡ 技巧：Git 别名自动化

让 Claude Code 帮你设置 Git 别名：

```bash
# 智能 commit：自动生成 message
git config alias.smart-commit '!f() { 
  msg=$(claude "根据 git diff 生成简洁的 commit message，只要一行"); 
  git commit -m "$msg"; 
}; f'

# 使用
git add .
git smart-commit
```

## 📝 实战案例五：代码审查准备

### 需求描述

```
我要提交 PR 了，帮我整理改动说明
```

### Claude Code 执行

```bash
# 对比主分支
git log main..HEAD --oneline
git diff main...HEAD --stat
```

**生成的 PR 描述：**

```markdown
## 改动概述
本次更新添加了用户角色系统，优化了认证流程。

## 主要改动

### 新功能
- 用户角色字段（admin/user/guest）
- 角色权限中间件
- 用户角色管理 API

### 改进
- 认证逻辑重构，性能提升 30%
- 错误提示更友好

### 测试
- 新增 12 个测试用例
- 测试覆盖率从 78% 提升到 85%

## 相关 Issue
- Closes #45 用户角色功能需求
- Closes #67 认证性能优化

## 测试方法
```bash
npm run test
npm run test:e2e
```

## Checklist
- [x] 代码符合项目规范
- [x] 添加了必要的测试
- [x] 文档已更新
- [x] 无 TypeScript 错误
```

## 🎓 学到了什么

1. **Commit message 让 Claude Code 写，规范又省心**
2. **大改动分批提交，历史更清晰**
3. **冲突解决可以自动化**
4. **PR 描述让 Claude Code 整理，专业又完整**

## ⚠️ 注意

- 重要操作先让 Claude Code 解释要做什么，确认后再执行
- 提交前检查一下改动是否正确
- 不要把敏感文件提交进去
