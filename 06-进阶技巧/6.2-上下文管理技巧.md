# 6.2 上下文管理技巧

> 让 Claude Code 记住重要信息，写出更准确的代码。

## 🎯 场景描述

Claude Code 有上下文限制，对话太长会"忘记"之前的内容。你需要让它记住：
- 项目的技术栈
- 代码规范
- 常用的配置

## 💡 核心思路

```
用 CLAUDE.md / .cursorrules 等文件持久化上下文
```

## 📝 方法一：CLAUDE.md 文件

在项目根目录创建 `CLAUDE.md`：

```markdown
# 项目上下文

## 技术栈
- 后端：Node.js + Express + TypeScript
- 数据库：PostgreSQL + Prisma
- 前端：React + Tailwind CSS
- 测试：Jest

## 代码规范
- 使用 async/await，不用 .then()
- 错误处理用 try-catch
- API 响应格式：{ success: boolean, data?: any, error?: string }

## 常用命令
- 开发：npm run dev
- 测试：npm test
- 数据库迁移：npx prisma migrate dev

## 重要文件
- src/routes/ - API 路由
- src/services/ - 业务逻辑
- src/types/ - 类型定义
```

Claude Code 会自动读取这个文件，知道你的项目背景。

## 📝 方法二：.cursorrules 文件

适用于 Cursor IDE 用户：

```
# .cursorrules
- 所有代码用 TypeScript
- 使用函数式组件 + Hooks
- CSS 用 Tailwind，不用 styled-components
- 每个组件要有对应的测试文件
- 注释用中文
```

## 📝 方法三：自定义指令

在 Claude Code 中设置自定义指令（Settings → Custom Instructions）：

```
你是一个专业的 Node.js 开发者。

写代码时：
1. 优先使用 ES Modules（import/export）
2. 错误处理要完善
3. 变量命名要语义化
4. 添加必要的注释

不要：
1. 不要用 var
2. 不要省略错误处理
3. 不要用 any 类型
```

## 📝 方法四：对话中提供上下文

### 错误示范

```
帮我写一个登录接口
```

Claude Code 会问：用什么框架？什么数据库？返回什么格式？

### 正确示范

```
项目背景：
- Express + TypeScript
- 用户数据在 PostgreSQL，用 Prisma ORM
- 密码用 bcrypt 加密
- 返回 JWT token

帮我写一个登录接口：
POST /auth/login
Body: { email, password }
返回: { success: true, token: "xxx" }
```

## ⚡ 技巧：分块提供大文件

文件太大 Claude Code 一次读不完？

```
先读第一部分：
请读取 src/handlers/userHandler.ts 的前 100 行

然后继续：
请读取同一文件的 101-200 行

让它处理：
现在帮我重构这个文件的 handleCreateUser 函数
```

## 📝 实战案例：新项目快速开始

### 第一步：让 Claude Code 帮你创建 CLAUDE.md

```
我刚创建了一个 Next.js 项目，请帮我：
1. 分析项目结构
2. 生成一个 CLAUDE.md 文件，包含技术栈、代码规范、常用命令
```

### 第二步：Claude Code 自动生成

```markdown
# CLAUDE.md

## 技术栈
- 框架：Next.js 14 (App Router)
- 语言：TypeScript
- 样式：Tailwind CSS
- 数据获取：TanStack Query
- 表单：React Hook Form + Zod

## 目录结构
- app/ - 页面路由
- components/ - 组件
- lib/ - 工具函数
- types/ - 类型定义

## 代码规范
- 组件用 'use client' 指令区分客户端组件
- API 路由放在 app/api/
- 样式优先用 Tailwind class

## 常用命令
- dev: npm run dev
- build: npm run build
- lint: npm run lint
```

以后每次对话，Claude Code 都知道这个项目的情况了！

## 🎓 学到了什么

1. **CLAUDE.md 是给 Claude Code 看的项目说明书**
2. **提供充分上下文，代码质量更高**
3. **新项目先建 CLAUDE.md，以后省很多事**
4. **大文件分块读取，不要一次塞太多**

## ⚠️ 注意

- CLAUDE.md 不要写敏感信息（API Key 等）
- 定期更新，保持和项目同步
- 不要写太多，太长反而会被截断
