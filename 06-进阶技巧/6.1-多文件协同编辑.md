# 6.1 多文件协同编辑

> 让 Claude Code 同时改多个文件，保持一致性。

## 🎯 场景描述

你要重构一个功能，涉及多个文件：
- 修改 API 接口定义
- 更新调用方的代码
- 同步更新测试文件
- 更新文档

一个一个改太慢，还容易漏。Claude Code 可以帮你一起改。

## 💡 核心思路

```
告诉 Claude Code 变更范围 + 让它同时修改所有相关文件
```

## 📝 实战案例一：重命名函数

### 需求描述

```
我要把项目里的 getUserById 改成 findUserById，涉及：
- services/userService.js
- controllers/userController.js  
- tests/user.test.js
- docs/api.md

帮我一起改，不要漏掉任何地方。
```

### Claude Code 执行的操作

```javascript
// services/userService.js - 改前
async function getUserById(id) {
  return await db.users.findUnique({ where: { id } });
}

// services/userService.js - 改后
async function findUserById(id) {
  return await db.users.findUnique({ where: { id } });
}
```

```javascript
// controllers/userController.js - 改前
const user = await getUserById(req.params.id);

// controllers/userController.js - 改后
const user = await findUserById(req.params.id);
```

```javascript
// tests/user.test.js - 改前
const result = await getUserById(1);

// tests/user.test.js - 改后
const result = await findUserById(1);
```

```markdown
<!-- docs/api.md - 改前 -->
- `GET /users/:id` - 调用 `getUserById`

<!-- docs/api.md - 改后 -->
- `GET /users/:id` - 调用 `findUserById`
```

## 📝 实战案例二：添加新字段

### 需求描述

```
给 User 添加一个 role 字段，涉及：

1. 数据库 schema (schema.prisma)
2. 类型定义 (types/user.ts)
3. 服务层 (services/userService.ts)
4. API 响应 (controllers/userController.ts)
5. 前端组件 (components/UserCard.vue)
6. 测试数据 (tests/fixtures/user.json)

role 的值：'admin' | 'user' | 'guest'
默认值：'user'
```

### Claude Code 一键搞定

**schema.prisma**
```prisma
model User {
  id        Int      @id @default(autoincrement())
  email     String   @unique
  name      String
  role      String   @default("user")  // 新增
  createdAt DateTime @default(now())
}
```

**types/user.ts**
```typescript
export interface User {
  id: number;
  email: string;
  name: string;
  role: 'admin' | 'user' | 'guest';  // 新增
  createdAt: Date;
}
```

**services/userService.ts**
```typescript
async function createUser(data: CreateUserData) {
  return await db.user.create({
    data: {
      ...data,
      role: data.role || 'user',  // 默认值
    }
  });
}
```

**tests/fixtures/user.json**
```json
{
  "id": 1,
  "email": "test@example.com",
  "name": "Test User",
  "role": "user"
}
```

## 📝 实战案例三：迁移到新 API

### 需求描述

```
项目要从 REST API 迁移到 GraphQL，涉及：

1. 替换所有 fetch('/api/xxx') 为 GraphQL 查询
2. 更新类型定义
3. 修改错误处理逻辑
4. 更新 mock 数据

帮我处理 src/api/ 目录下所有文件。
```

### Claude Code 批量转换

**改前 (REST)**
```typescript
// api/users.ts
export async function getUsers() {
  const res = await fetch('/api/users');
  return res.json();
}

export async function getUser(id: number) {
  const res = await fetch(`/api/users/${id}`);
  return res.json();
}
```

**改后 (GraphQL)**
```typescript
// api/users.ts
import { gql, client } from './client';

export async function getUsers() {
  const { data } = await client.query({
    query: gql`
      query GetUsers {
        users {
          id
          name
          email
        }
      }
    `
  });
  return data.users;
}

export async function getUser(id: number) {
  const { data } = await client.query({
    query: gql`
      query GetUser($id: Int!) {
        user(id: $id) {
          id
          name
          email
        }
      }
    `,
    variables: { id }
  });
  return data.user;
}
```

## ⚡ 技巧：让 Claude Code 自己找关联文件

```
我要把项目里的 moment.js 换成 dayjs。
请先找出所有使用 moment.js 的文件，然后一起修改。
```

Claude Code 会：
1. 搜索 `import moment` 或 `require('moment')`
2. 找出所有使用的地方
3. 批量替换为 dayjs
4. 处理 API 差异（moment 和 dayjs 的 API 不完全一样）

## 🎓 学到了什么

1. **多文件修改要一次性描述清楚，让 Claude Code 全部处理**
2. **改名、加字段、迁移 API 都可以自动化**
3. **让 Claude Code 先搜索再改，不会遗漏**
4. **改完后跑一遍测试验证**

## ⚠️ 注意事项

- 改动前先提交代码（方便回滚）
- 大范围改动让 Claude Code 列出文件清单确认
- 改完后跑测试确保没破坏功能
