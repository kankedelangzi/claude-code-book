# 2.4 配置文件管理

> 让 Claude Code 帮你生成、管理、迁移配置文件，告别手动复制粘贴。

## 🎯 场景描述

项目配置是个麻烦事：

- 新项目不知道用什么配置
- 配置在本地改了但没同步到团队
- 配置文件格式记不住（YAML 缩进、JSON 语法）
- 迁移项目时配置要重写一遍

**预计耗时：30 分钟到 2 小时**

Claude Code 可以：
- 根据你的需求生成配置
- 把配置转换成不同格式
- 检查配置错误
- 批量更新多个配置文件

## ✨ Claude Code 做法

### 案例 1：生成项目配置

直接描述你需要的配置：

```
在当前目录生成一个 Node.js 项目的配置文件：

1. package.json：
   - 项目名：my-api-server
   - 描述：RESTful API 服务
   - 主入口：src/index.ts
   - scripts：
     * dev：用 ts-node-dev 运行
     * build：编译到 dist
     * start：运行编译后的代码
     * test：运行 jest 测试
   - 依赖：express, cors, helmet, dotenv
   - 开发依赖：typescript, @types/*, ts-node-dev, jest, ts-jest

2. tsconfig.json：
   - 目标：ES2020
   - 模块：commonjs
   - 严格模式开启
   - 输出目录：dist
   - 源码目录：src

3. .env.example：
   - PORT=3000
   - NODE_ENV=development
   - DATABASE_URL=
   - LOG_LEVEL=info

4. .gitignore：
   - node_modules
   - dist
   - .env
   - *.log
   - coverage
```

Claude Code 会生成完整的配置文件。

### 案例 2：配置格式转换

你有一段 JSON 配置，想转成 YAML 或 TOML：

```
把以下 JSON 配置转成 YAML 格式：

{
  "database": {
    "host": "localhost",
    "port": 5432,
    "name": "myapp",
    "user": "admin",
    "password": "secret",
    "pool": {
      "min": 2,
      "max": 10
    }
  },
  "redis": {
    "host": "127.0.0.1",
    "port": 6379,
    "db": 0
  }
}
```

Claude Code 会生成：

```yaml
database:
  host: localhost
  port: 5432
  name: myapp
  user: admin
  password: secret
  pool:
    min: 2
    max: 10

redis:
  host: 127.0.0.1
  port: 6379
  db: 0
```

### 案例 3：批量更新配置

多个服务需要更新相同的配置项：

```
当前目录有多个 docker-compose 文件：
- docker-compose.yml
- docker-compose.dev.yml
- docker-compose.prod.yml

给每个文件的 services 添加环境变量：
  RESTART: unless-stopped
  LOG_LEVEL: ${LOG_LEVEL:-info}

使用环境变量，默认值 info。
```

### 案例 4：生成 Nginx 配置

```
生成一个 Nginx 反向代理配置：

需求：
- 监听 80 端口
- 静态文件服务：/var/www/html
- API 代理到 localhost:3000
- WebSocket 代理到 localhost:3001
- 启用 gzip 压缩
- 设置缓存策略：静态资源缓存 7 天
- 添加安全头：X-Frame-Options, CSP, HSTS
```

Claude Code 会生成：

```nginx
server {
    listen 80;
    server_name _;
    root /var/www/html;
    index index.html;

    # 安全头
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # 静态文件
    location / {
        try_files $uri $uri/ =404;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # API 代理
    location /api/ {
        proxy_pass http://localhost:3000/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_cache_bypass $http_upgrade;
    }

    # WebSocket 代理
    location /ws/ {
        proxy_pass http://localhost:3001/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
    }

    # 启用 gzip
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css text/xml text/javascript application/javascript application/json application/xml;
}
```

## 🚀 常用配置模板

### Docker 配置

```dockerfile
# 多阶段构建的 Node.js 应用
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

FROM node:20-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./

USER node
EXPOSE 3000
HEALTHCHECK --interval=30s --timeout=3s CMD wget --no-verbose --tries=1 --spider http://localhost:3000/health || exit 1

CMD ["node", "dist/index.js"]
```

### GitHub Actions 配置

```yaml
name: CI

on:

    branches: [main]
  pull_request:
    branches: [main]

          name: dist
```

## 💡 配置管理技巧

### 1. 环境变量命名规范

```
# ✅ 好的命名
DATABASE_HOST
DATABASE_PORT
DATABASE_NAME
REDIS_URL
API_KEY

# ❌ 不好的命名
DB_HOST
PORT
dbname
api-key
```

### 2. 使用 .env.example 而不是 .env

```bash
# .env.example（提交到 Git）
# 填上默认值或占位符
DATABASE_URL=postgres://localhost:5432/myapp
PORT=3000
LOG_LEVEL=info

# .env（不提交到 Git）
# 填上真实值
DATABASE_URL=postgres://user:pass@prod-host:5432/production
PORT=3001
LOG_LEVEL=debug
```

### 3. 配置分层

```
config/
├── default.yaml      # 默认配置
├── development.yaml  # 开发环境
├── production.yaml   # 生产环境
└── local.yaml        # 本地覆盖（不提交）

使用配置加载器合并：
const config = { 
  ...default, 
  ...envConfig, 
  ...local 
};
```

## ⚠️ 注意事项

1. **敏感信息不要提交**——.env、secrets.yml 等用 .gitignore 排除
2. **配置格式要校验**——提交前用对应工具验证语法
3. **文档和配置保持一致**——README 里的配置说明要同步更新
4. **版本控制配置变更**——配置修改也要 code review

## 📝 小结

配置文件虽然不是业务代码，但直接影响项目的可维护性。Claude Code 可以：

- 快速生成标准配置
- 格式转换不用查文档
- 批量更新省去重复劳动
- 生成配置模板避免每次从零开始

**好配置的标准：别人能看懂、本地能覆盖、不提交敏感信息。**
