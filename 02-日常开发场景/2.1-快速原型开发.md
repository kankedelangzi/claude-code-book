# 2.1 快速原型开发

> 想法 → 可运行原型，从小时级缩短到分钟级。

## 🎯 场景描述

你有一个产品想法，需要快速验证可行性。传统流程：

1. 写需求文档
2. 设计架构
3. 搭建项目骨架
4. 实现核心功能
5. 跑起来看效果

**预计耗时：1-3 天**

而且大部分时间花在"搭架子"——配置开发环境、装依赖、写脚手架代码。

## ✨ Claude Code 做法

你只需要说清楚要做什么：

```
帮我快速搭一个原型：

目标：验证「AI 自动生成周报」的想法
功能：
- 用户输入本周完成的任务（多条文本）
- 调用 AI API 生成周报摘要
- 预览并支持微调

技术栈：
- 前端：React + Tailwind
- 后端：Node.js + Express
- AI：直接调用 Claude API

要求：
- 能跑起来就行，不追求完美
- 单仓库，前后端在一起
- 带一个简单的 README 说明怎么启动
```

**Claude Code 会自动：**

1. 创建项目结构
2. 生成 `package.json` 和依赖配置
3. 写前端组件和页面
4. 写后端 API 路由
5. 配置启动脚本
6. 生成 README

**预计耗时：10-20 分钟**

## 🚀 启动验证

```bash
cd ai-weekly-report
npm install
npm run dev
# 打开 http://localhost:3000
```

**立刻能看到原型效果**——这就是快速验证的意义。

## 🔧 原型迭代：从"能跑"到"好用"

原型跑起来后，你可以边用边改：

### 第一轮：发现输入体验差

```
任务输入体验不好，改成：
- 支持粘贴多行，自动拆分成多条任务
- 每条任务可以标记优先级（高/中/低）
- 加一个「清空」按钮
```

### 第二轮：想看生成过程

```
周报生成时加个 loading 动画，
生成完成后有「复制到剪贴板」按钮。
```

### 第三轮：要保存历史

```
加个历史记录功能：
- 本地 localStorage 存最近 10 条
- 可以查看、重新加载、删除
```

**每一轮都是几分钟的事**——不需要重新规划、不需要重构，直接改。

## 💡 快速原型的核心思维

### 1. 先跑起来，再优化

传统开发思维：设计 → 实现 → 测试 → 上线
原型思维：**跑起来 → 用起来 → 改起来**

Claude Code 最适合这种"边用边改"的模式。

### 2. 用自然语言描述，让 AI 补细节

你不需要知道：
- React 项目怎么配置
- Express 路由怎么写
- Tailwind 类名有哪些

你只需要说：
- "要一个输入框"
- "点按钮后调用 API"
- "结果显示在右边"

### 3. 原型 ≠ 最终产品

原型是用来验证想法的，不是用来上线的。接受这些"不完美"：

- 代码结构可能不够优雅
- 没有完整的错误处理
- 样式是"能用就行"
- 没有测试

**验证通过后再重构**——那时候你才知道哪些设计是必要的。

## 🛠️ 常见原型场景

### 场景 1：验证 API 集成

```
帮我写个脚本测试这个 API：
- 调用 https://api.example.com/users
- 传参数 page=1, limit=10
- 打印返回的 JSON
- 计算响应时间
```

### 场景 2：数据转换工具

```
写个工具：
- 读取 CSV 文件
- 按某列分组统计
- 输出新的汇总 CSV
- 支持命令行参数指定输入输出文件
```

### 场景 3：简单 Web 界面

```
搭个单页应用：
- 左边是表单，右边是结果
- 表单有 5 个输入项
- 提交后调用后端计算
- 结果以表格形式展示
```

### 场景 4：命令行工具

```
写个 CLI 工具：
- 命令名：mytool
- 子命令：init, run, clean
- init 创建配置文件
- run 执行主逻辑
- clean 清理临时文件
```

## ⚠️ 注意事项

1. **原型要标注"仅供参考"**——别让团队误以为是正式代码
2. **核心逻辑自己 review**——AI 可能理解错需求
3. **敏感信息别写死**——API key 等用环境变量
4. **原型废弃要果断**——验证失败就删掉，别恋战

## 📝 小结

快速原型的本质是**降低验证成本**。Claude Code 让你：

- 不用花时间搭骨架
- 不用查文档写配置
- 不用纠结技术细节
- 专注于"这个想法值不值得做"

**想法廉价，验证昂贵。Claude Code 让验证也变得廉价。**
