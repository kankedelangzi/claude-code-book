## 🎯 场景描述

数据库操作是后端开发的基础，但写起来枯燥：

- 每个表都要写 CRUD，太重复
- SQL 语法记不住，参数绑定容易出错
- 数据库迁移脚本写起来麻烦
- 联表查询的 SQL 总是写错

**预计耗时：1-3 小时**

Claude Code 可以：
- 根据数据模型生成完整的 CRUD 代码
- 写数据库迁移脚本
- 生成复杂的联表查询 SQL
- 帮你做数据库连接池配置

```bash
模型：

   - product_id: Integer 外键
   - quantity: Integer 数量
   - unit_price: Float 下单时的单价

生成完整的 models.py，包含所有模型定义和关系配置。
```

Claude Code 生成：

```python
# models.py
from datetime import datetime
from sqlalchemy import create_engine, Column, Integer, String, Float, DateTime, ForeignKey
from sqlalchemy.orm import declarative_base, relationship, sessionmaker

Base = declarative_base()
engine = create_engine('sqlite:///shop.db', echo=True)
SessionLocal = sessionmaker(bind=engine)

class Category(Base):
    __tablename__ = 'categories'
    
    id = Column(Integer, primary_key=True, autoincrement=True)
    name = Column(String(50), nullable=False)
    
    products = relationship('Product', back_populates='category')

class Product(Base):
    __tablename__ = 'products'
    
    id = Column(Integer, primary_key=True, autoincrement=True)
    name = Column(String(100), nullable=False)
    price = Column(Float, nullable=False)
    stock = Column(Integer, default=0)
    category_id = Column(Integer, ForeignKey('categories.id'))
    created_at = Column(DateTime, default=datetime.utcnow)
    
    category = relationship('Category', back_populates='products')
    order_items = relationship('OrderItem', back_populates='product')

class OrderItem(Base):
    __tablename__ = 'order_items'
    
    id = Column(Integer, primary_key=True, autoincrement=True)
    order_id = Column(Integer, ForeignKey('orders.id'))
    product_id = Column(Integer, ForeignKey('products.id'))
    quantity = Column(Integer, nullable=False)
    unit_price = Column(Float, nullable=False)
    
    product = relationship('Product', back_populates='order_items')

# 创建所有表
Base.metadata.create_all(bind=engine)
```

### 案例 3：手写 SQL 迁移脚本

```bash
# 为 MySQL 数据库写一个用户表的迁移脚本：

1. 创建 users 表：
   - id: BIGINT 自增主键
   - email: VARCHAR(255) 唯一索引
   - username: VARCHAR(50) 唯一索引
   - password_hash: VARCHAR(255)
   - status: TINYINT 默认 1（1=正常，0=禁用）
   - created_at: DATETIME 默认当前时间
   - updated_at: DATETIME 自动更新

2. 创建索引：
   - email 唯一索引
   - username 唯一索引
   - status 普通索引

3. 添加备注：
   - 表备注：用户表
   - email 备注：登录邮箱

4. 同时写回滚脚本（DROP TABLE）

用事务包裹，确保原子性。
```

生成：

```sql
-- up.sql
-- 创建用户表
CREATE TABLE IF NOT EXISTS `users` (
  `id` BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `email` VARCHAR(255) NOT NULL COMMENT '登录邮箱',
  `username` VARCHAR(50) NOT NULL COMMENT '用户名',
  `password_hash` VARCHAR(255) NOT NULL COMMENT '密码哈希',
  `status` TINYINT NOT NULL DEFAULT 1 COMMENT '状态：1=正常，0=禁用',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_email` (`email`),
  UNIQUE KEY `uk_username` (`username`),
  KEY `idx_status` (`status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户表';

-- down.sql
-- 回滚：删除用户表
DROP TABLE IF EXISTS `users`;
```

### 案例 5：复杂查询 SQL

```bash
# 生成一个统计报表的 SQL（MySQL）：

需求：统计每个月的销售数据

输出字段：
- month: 月份（YYYY-MM）
- total_orders: 订单总数
- total_amount: 销售总额
- avg_order_value: 客单价
- new_customers: 新客户数（当月首次下单）
- repeat_customers: 复购客户数

表结构：
- orders: id, customer_id, total_amount, status, created_at
- customers: id, created_at（注册时间）

条件：
- 只统计已支付订单（status = 'paid'）
- 时间范围：最近 12 个月
- 按月份降序排列
```

生成：

```sql
SELECT 
    DATE_FORMAT(o.created_at, '%Y-%m') AS `month`,
    COUNT(DISTINCT o.id) AS total_orders,
    SUM(o.total_amount) AS total_amount,
    ROUND(AVG(o.total_amount), 2) AS avg_order_value,
    COUNT(DISTINCT CASE 
        WHEN c.created_at >= DATE_FORMAT(o.created_at, '%Y-%m-01')
        AND c.created_at < DATE_FORMAT(DATE_ADD(o.created_at, INTERVAL 1 MONTH), '%Y-%m-01')
        THEN c.id 
    END) AS new_customers,
    COUNT(DISTINCT CASE 
        WHEN (SELECT COUNT(*) FROM orders o2 
              WHERE o2.customer_id = o.customer_id 
              AND DATE_FORMAT(o2.created_at, '%Y-%m') < DATE_FORMAT(o.created_at, '%Y-%m')) > 0
        THEN c.id 
    END) AS repeat_customers
FROM orders o
JOIN customers c ON o.customer_id = c.id
WHERE o.status = 'paid'
  AND o.created_at >= DATE_SUB(CURDATE(), INTERVAL 12 MONTH)
GROUP BY DATE_FORMAT(o.created_at, '%Y-%m')
ORDER BY `month` DESC;
```

## 🚀 常用数据库配置模板

### PostgreSQL 连接配置

```typescript
// config/database.ts
import { Pool } from 'pg';

export const pool = new Pool({
  host: process.env.DB_HOST || 'localhost',
  port: parseInt(process.env.DB_PORT || '5432'),
  database: process.env.DB_NAME,
  user: process.env.DB_USER,
  password: process.env.DB_PASSWORD,
  max: 20,           // 最大连接数
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000,
});

pool.on('error', (err) => {
  console.error('Unexpected database error:', err);
});
```

### Redis 缓存配置

```typescript
// config/cache.ts
import Redis from 'ioredis';

export const redis = new Redis({
  host: process.env.REDIS_HOST || 'localhost',
  port: parseInt(process.env.REDIS_PORT || '6379'),
  password: process.env.REDIS_PASSWORD,
  db: parseInt(process.env.REDIS_DB || '0'),
  keyPrefix: 'app:',
  retryStrategy(times) {
    const delay = Math.min(times * 50, 2000);
    return delay;
  },
});

redis.on('error', (err) => console.error('Redis error:', err));
```

## 💡 数据库操作技巧

### 1. 始终使用参数化查询

```sql
-- ✅ 安全：参数化查询
SELECT * FROM users WHERE email = $1;

-- ❌ 危险：字符串拼接 SQL
SELECT * FROM users WHERE email = '" + email + "';
```

### 2. 批量操作用事务

```sql
BEGIN;
INSERT INTO orders (customer_id, total) VALUES (1, 100);
INSERT INTO order_items (order_id, product_id, qty) VALUES (currval('orders_id_seq'), 1, 2);
INSERT INTO order_items (order_id, product_id, qty) VALUES (currval('orders_id_seq'), 2, 1);
COMMIT;
```

### 3. 分页查询规范

```sql
-- MySQL 分页
SELECT * FROM posts 
ORDER BY created_at DESC 
LIMIT 20 OFFSET 40;  -- 第 3 页，每页 20 条

-- PostgreSQL 分页（推荐）
SELECT * FROM posts 
ORDER BY created_at DESC 
LIMIT 20 OFFSET 40;

-- 更高效的游标分页（适合大表）
SELECT * FROM posts 
WHERE id < #{last_id}
ORDER BY id DESC 
LIMIT 20;
```

## ⚠️ 注意事项

1. **密码必须哈希**——永远不要明文存储密码
2. **敏感数据要加密**——身份证号、银行卡号等用 AES 加密存储
3. **大表加索引**——根据查询条件添加合适的索引
4. **迁移要备份**——生产环境执行前先备份数据
5. **连接池要配置**——避免数据库连接耗尽
