# 2.2 API 接口编写

> 从接口文档到可运行的 API，原来可以这么快。

## 🎯 场景描述

你需要写一组 API 接口，传统流程是：

1. 设计接口文档（路径、参数、返回值）
2. 搭建项目结构
3. 写路由和控制器
4. 实现参数校验
5. 写错误处理
6. 测试接口

**预计耗时：半天到一天**

大部分时间在写**重复性代码**——每个接口都要处理类似的校验、错误、返回格式。

## ✨ Claude Code 做法

直接描述你要的接口：

```
帮我写一组用户管理 API，Node.js + Express：

接口列表：
1. GET /api/users - 获取用户列表（支持分页 page/limit，支持按 name 搜索）
2. GET /api/users/:id - 获取单个用户详情
3. POST /api/users - 创建用户（name, email, age 必填，email 要格式校验）
4. PUT /api/users/:id - 更新用户信息
5. DELETE /api/users/:id - 删除用户

要求：
- 用内存数组模拟数据库（先不用连真实数据库）
- 统一的返回格式：{ success: boolean, data: any, message?: string }
- 参数校验失败返回 400，资源不存在返回 404
- 每个接口都要有简单的测试示例
- 生成 README 说明怎么启动和测试
```

## 📋 生成的代码示例

### server.js - 入口文件

```javascript
const express = require('express');
const userRoutes = require('./routes/users');

const app = express();
const PORT = 3000;

// 中间件
app.use(express.json());

// 路由
app.use('/api/users', userRoutes);

// 404 处理
app.use((req, res) => {
  res.status(404).json({ success: false, message: '接口不存在' });
});

// 错误处理
app.use((err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ success: false, message: '服务器内部错误' });
});

app.listen(PORT, () => {
  console.log(`Server running on http://localhost:${PORT}`);
});
```

### routes/users.js - 用户路由

```javascript
const express = require('express');
const router = express.Router();
const { validateUser } = require('../middleware/validator');

// 模拟数据库
let users = [
  { id: 1, name: '张三', email: 'zhangsan@example.com', age: 25 },
  { id: 2, name: '李四', email: 'lisi@example.com', age: 30 }
];
let nextId = 3;

// GET /api/users - 获取用户列表
router.get('/', (req, res) => {
  const { page = 1, limit = 10, name } = req.query;
  
  let result = [...users];
  
  // 搜索过滤
  if (name) {
    result = result.filter(u => u.name.includes(name));
  }
  
  // 分页
  const start = (page - 1) * limit;
  const end = start + parseInt(limit);
  const paginated = result.slice(start, end);
  
  res.json({
    success: true,
    data: {
      list: paginated,
      total: result.length,
      page: parseInt(page),
      limit: parseInt(limit)
    }
  });
});

// GET /api/users/:id - 获取单个用户
router.get('/:id', (req, res) => {
  const user = users.find(u => u.id === parseInt(req.params.id));
  
  if (!user) {
    return res.status(404).json({ success: false, message: '用户不存在' });
  }
  
  res.json({ success: true, data: user });
});

// POST /api/users - 创建用户
router.post('/', validateUser, (req, res) => {
  const { name, email, age } = req.body;
  
  // 检查 email 是否已存在
  if (users.some(u => u.email === email)) {
    return res.status(400).json({ success: false, message: '邮箱已被使用' });
  }
  
  const newUser = { id: nextId++, name, email, age };
  users.push(newUser);
  
  res.status(201).json({ success: true, data: newUser });
});

// PUT /api/users/:id - 更新用户
router.put('/:id', validateUser, (req, res) => {
  const user = users.find(u => u.id === parseInt(req.params.id));
  
  if (!user) {
    return res.status(404).json({ success: false, message: '用户不存在' });
  }
  
  const { name, email, age } = req.body;
  
  // 检查 email 是否被其他用户使用
  if (email && email !== user.email && users.some(u => u.email === email)) {
    return res.status(400).json({ success: false, message: '邮箱已被使用' });
  }
  
  Object.assign(user, { name, email, age });
  res.json({ success: true, data: user });
});

// DELETE /api/users/:id - 删除用户
router.delete('/:id', (req, res) => {
  const index = users.findIndex(u => u.id === parseInt(req.params.id));
  
  if (index === -1) {
    return res.status(404).json({ success: false, message: '用户不存在' });
  }
  
  const deleted = users.splice(index, 1)[0];
  res.json({ success: true, data: deleted });
});

module.exports = router;
```

### middleware/validator.js - 参数校验

```javascript
function validateUser(req, res, next) {
  const { name, email, age } = req.body;
  const errors = [];
  
  // 创建时必须传所有字段
  if (req.method === 'POST') {
    if (!name) errors.push('name 必填');
    if (!email) errors.push('email 必填');
    if (age === undefined) errors.push('age 必填');
  }
  
  // email 格式校验
  if (email && !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email)) {
    errors.push('email 格式不正确');
  }
  
  // age 必须是数字
  if (age !== undefined && (typeof age !== 'number' || age < 0)) {
    errors.push('age 必须是正整数');
  }
  
  if (errors.length > 0) {
    return res.status(400).json({ success: false, message: errors.join(', ') });
  }
  
  next();
}

module.exports = { validateUser };
```

## 🔧 迭代优化

### 第一轮：加真实数据库

```
把内存数组换成 SQLite：
- 用 sqlite3 包
- 启动时自动建表
- 保留现有接口格式不变
```

### 第二轮：加 JWT 认证

```
给 API 加上登录认证：
- POST /api/login - 用户名密码登录，返回 token
- 其他接口需要 Authorization: Bearer <token>
- 用 jsonwebtoken 包
```

### 第三轮：加接口文档

```
集成 Swagger/OpenAPI：
- 访问 /api-docs 能看到接口文档
- 可以直接在页面上测试接口
```

**每轮迭代都是几分钟的事。**

## 💡 写好 API 的技巧

### 1. 先描述接口列表

把要哪些接口一次性说清楚，Claude Code 会帮你规划路由结构。

### 2. 说明返回格式

统一返回格式很重要，提前说明：

```
统一返回格式：{ success: boolean, data: any, message?: string }
```

### 3. 先 mock，后接真实数据库

初期用内存数组或 JSON 文件模拟，验证接口设计没问题后，再换成真实数据库。

### 4. 要测试示例

生成 `.http` 文件或 curl 命令，方便你立即测试。

## ⚠️ 注意事项

1. **生产环境要加限流**——Claude Code 生成的代码没有限流，上线前要加
2. **敏感操作要加日志**——删除、修改操作记录操作人
3. **数据库操作要加事务**——涉及多表更新时
4. **密码要哈希存储**——别明文存密码

**好的 API 设计在于清晰的约定，而不是代码量。**
